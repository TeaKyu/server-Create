# 01. 프로젝트 개요 및 아키텍처

## 프로젝트 목적

Windows PC의 VirtualBox + Vagrant 환경에서 Rocky Linux 9 기반 Kubernetes 1.34 클러스터를 구축하고,
운영에 필요한 전체 DevOps 스택을 설치하여 실제 운영 가능한 인프라를 만드는 것이 목표입니다.

---

## 클러스터 구성

| 노드 | 호스트명 | IP | CPU | RAM | 역할 |
|------|---------|-----|-----|-----|------|
| Master | k8s-master | 192.168.56.30 | 4 core | 8 GB | Control Plane + 도구 설치 |
| Worker 1 | k8s-worker1 | 192.168.56.31 | 2 core | 8 GB | 워크로드 실행 |
| Worker 2 | k8s-worker2 | 192.168.56.32 | 2 core | 8 GB | 워크로드 실행 |

총 RAM 사용: 24GB (호스트 OS 포함 최소 32GB PC 권장)

## 설치 순서 및 의존성

스크립트는 **4개 파일**로 분리되어 있으며, 실행 순서는 의존성을 고려하여 설계되었습니다.

```
[k8s-common.sh]      → 3대 모두: OS 설정 + containerd + kubeadm
[k8s-master.sh]      → Master: kubeadm init + Calico (클러스터 생성)
 ── 워커 조인 ──      → Worker 2대: kubeadm join (클러스터 합류)
[k8s-master-apps.sh] → Master: 전체 DevOps 스택 설치
```

워커가 먼저 조인되어야 스택의 Pod들이 워커에 스케줄링되므로,
**클러스터 초기화**와 **스택 설치**를 분리했습니다.

### 스택 설치 순서 (k8s-master-apps.sh 내부)

```
[6] Helm            → 이후 모든 Helm 설치의 전제조건
[7] MetalLB         → Envoy Gateway가 External IP를 받으려면 필요
[8] Envoy Gateway   → MetalLB에서 IP를 받아 Gateway API로 트래픽 수신
[9] cert-manager    → Gateway에 TLS를 걸려면 필요
[10] Headlamp     → Helm 필요
[11] Prometheus     → Helm 필요
[12] Loki           → Helm 필요, Grafana에서 데이터소스 연동
[13] Sealed Secrets → Helm 필요
[14] Argo CD        → Manifest 설치 (Helm 불필요)
[15] Image Updater  → Argo CD가 먼저 설치되어야 함
```

## 운영 환경 전환 시 변경 포인트

현재 구성은 VirtualBox 학습 환경에 최적화되어 있습니다.
운영 서버로 전환할 때는 아래 항목들을 변경합니다.
**Helm 차트가 동일하므로 values.yaml만 교체하면 됩니다.**

| 컴포넌트 | 학습 (현재) | 운영 변경 |
|---------|-----------|----------|
| **Loki** | SingleBinary 1개 + emptyDir | SingleBinary 2개 + S3/GCS 스토리지 |
| **Alloy** | 기본 설정 (전체 수집) | 라벨 최적화, 불필요 로그 필터링, rate limit |
| **Prometheus** | retention 7일 | retention + 원격 쓰기(Thanos/Mimir) 또는 PV 확보 |
| **Grafana** | admin1234 | OIDC/SSO 연동, 비밀번호 변경 |
| **ArgoCD** | insecure 모드 (HTTP) | TLS 활성화 (cert-manager 연동) |
| **Headlamp** | session-ttl 패치 | 차트 업데이트 시 패치 불필요할 수 있음 |
| **Sealed Secrets** | 기본 | 공개키 백업 필수, 키 로테이션 정책 |
| **MetalLB** | Host-Only IP 대역 | 실제 네트워크 IP 대역 |
| **cert-manager** | self-signed | Let's Encrypt 또는 내부 CA |
| **노드** | VirtualBox 3대 (8GB) | 물리/클라우드 서버, 최소 3 worker 권장 |

> 각 컴포넌트의 운영 values 예시는 05-컴포넌트별-사용법-가이드.md에서 확인하세요.
