# 03. 파일 볼륨 설정 가이드 (첨부파일 PV/PVC)

## 개요

insideinfo-api 는 첨부파일을 서버 로컬 디스크에 저장합니다.  
k3s 파드는 재시작 시 컨테이너 스토리지가 초기화되므로, **호스트 경로를 PV/PVC로 마운트**해야 파일이 영구 보존됩니다.

| 환경 | 컨테이너 내부 마운트 경로 | 호스트 실제 경로 |
|------|--------------------------|----------------|
| dev  | `/app/upload/file`       | `/root/insideHomepage/upload/dev/file`  |
| prod | `/app/upload/file`       | `/root/insideHomepage/upload/prod/file` |

---

## 1단계 — 노드 Hostname 확인

PV는 특정 노드에 고정됩니다. 먼저 실제 노드 이름을 확인합니다.

```bash
kubectl get nodes
```

예시 출력:
```
NAME          STATUS   ROLES                  AGE
k3s-master    Ready    control-plane,master   10d
k3s-worker1   Ready    <none>                 5d
```

values 파일의 `persistence.nodeNames` 항목을 실제 hostname 으로 수정합니다.

**dev** (`insideinfo-gitops/apps/insideinfo-api/helm/values/dev/inside-api-values.yaml`):
```yaml
persistence:
  nodeNames:
    - k3s-master   # ← 실제 hostname 으로 변경
```

**prod** (`insideinfo-gitops/apps/insideinfo-api/helm/values/prod/inside-api-values.yaml`):
```yaml
persistence:
  nodeNames:
    - k3s-master   # ← 실제 hostname 으로 변경
```

## 2단계 — 호스트 디렉토리 생성

파일이 저장될 노드에 디렉토리를 미리 생성합니다.

```bash
# dev 디렉토리
mkdir -p /root/insideHomepage/upload/dev/file/data
mkdir -p /root/insideHomepage/upload/dev/file/editorImage
mkdir -p /root/insideHomepage/upload/dev/file/thumbnail

# prod 디렉토리
mkdir -p /root/insideHomepage/upload/prod/file/data
mkdir -p /root/insideHomepage/upload/prod/file/editorImage
mkdir -p /root/insideHomepage/upload/prod/file/thumbnail
```

---

## 3단계 — Helm/ArgoCD 배포

gitops 저장소에 변경사항을 push 하면 ArgoCD 가 자동 sync 합니다.  
수동으로 적용할 경우 아래 명령어를 사용합니다.

```bash
# dev 배포
helm upgrade --install insideinfo-api-dev \
  ./insideinfo-gitops/apps/insideinfo-api/helm \
  -f ./insideinfo-gitops/apps/insideinfo-api/helm/values/dev/inside-api-values.yaml \
  -n insideinfo

# prod 배포
helm upgrade --install insideinfo-api-prod \
  ./insideinfo-gitops/apps/insideinfo-api/helm \
  -f ./insideinfo-gitops/apps/insideinfo-api/helm/values/prod/inside-api-values.yaml \
  -n insideinfo
```

---

## 4단계 — PV/PVC 생성 확인

```bash
# dev 확인
kubectl get pv insideinfo-api-dev-upload-pv
kubectl get pvc insideinfo-api-dev-upload-pvc -n insideinfo

# prod 확인
kubectl get pv insideinfo-api-prod-upload-pv
kubectl get pvc insideinfo-api-prod-upload-pvc -n insideinfo
```

PVC 상태가 `Bound` 이어야 정상입니다.

## 5단계 — 기존 파일 이관 (구 서버 → k3s 노드)

기존 서버(`/home/design/insideHomepage/file/`)의 파일을 k3s 노드 호스트 경로로 복사합니다.

```bash
# 구 서버에서 실행 (rsync 이용)
rsync -avz /home/design/insideHomepage/file/ \
  root@<k3s-node-ip>:/root/insideHomepage/upload/prod/file/

# 또는 scp 이용
scp -r /home/design/insideHomepage/file/* \
  root@<k3s-node-ip>:/root/insideHomepage/upload/prod/file/
```

> **주의**: 파일 이관 후 DB의 FILE_PATH 경로도 마이그레이션이 필요합니다.  
> → 소스 변경 Plan (상대경로 저장 방식) 적용 후 아래 SQL을 실행합니다.

```sql
-- 기존 절대경로에서 /home/design/insideHomepage/file/ prefix 제거 → 상대경로로 변환
UPDATE ATCH_FILE
SET FILE_PATH = REPLACE(FILE_PATH, '/home/design/insideHomepage/file/', '')
WHERE FILE_PATH LIKE '/home/design/insideHomepage/file/%';

-- 변환 확인
SELECT ATCH_FILE_SN, FILE_PATH FROM ATCH_FILE LIMIT 20;
```

## 6단계 — 파드에서 마운트 확인

```bash
# 파드 이름 확인
kubectl get pods -n insideinfo

# 파드 내부에서 마운트 경로 확인
kubectl exec -it <pod-name> -n insideinfo -- ls -la /app/upload/file/

# 파드 볼륨 마운트 상세
kubectl describe pod <pod-name> -n insideinfo | grep -A 10 "Mounts:"
```

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| PVC 가 `Pending` 상태 | `nodeNames` hostname 불일치 | `kubectl get nodes` 로 정확한 hostname 확인 후 수정 |
| 파일 업로드 후 파드 재시작 시 파일 소멸 | PVC 미마운트 | `kubectl describe pod` 로 볼륨 마운트 상태 확인 |
| 파일 다운로드 404 | DB 경로 마이그레이션 미완료 | 5단계 마이그레이션 SQL 실행 |
| Permission denied | 호스트 디렉토리 권한 문제 | `chmod 777 /root/insideHomepage/upload/` |
