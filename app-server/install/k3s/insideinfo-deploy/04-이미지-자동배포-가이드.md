# 04. ArgoCD Image Updater 자동 배포 가이드

02-dev-prod-환경분리-가이드.md 완료 상태에서 ArgoCD Image Updater 를 활용해 **dev/prod 양쪽 모두** 이미지 태그를 자동 감지/반영하도록 설정하는 가이드.

- **dev**: Image Updater 가 감지 → gitops `dev` 브랜치 values 갱신 → ArgoCD `dev` App 이 **자동 Sync**
- **prod**: Image Updater 가 감지 → gitops `main` 브랜치 values 갱신 → ArgoCD `prod` App 은 **수동 Sync**

---

| 항목 | 상태 |

| 02-dev-prod-환경분리-가이드.md | 완료 |

| gitops 레포 | `main`, `dev` 브랜치 존재 |
| GitLab Container Registry | `gitlab.insideinfo.co.kr:5005` (자체 서명 인증서) |

## 동작 구조

Jenkins 가 브랜치별로 prefix 를 붙여 이미지 태그를 push 한다.
Image Updater 는 `allowTags` regex 로 `dev-*` / `prod-*` 를 구분해 각 환경의 values 파일만 갱신한다.

```
[dev 자동 배포]
Jenkins (BRANCH=dev)
  → docker push  <image>:dev-YYYYMMDD-HHmm
  → Image Updater (2분 폴링, regex ^dev-...)
    → gitops dev 브랜치 values/dev/*-values.yaml 의 image.tag 갱신 (git commit & push)
  → ArgoCD insideinfo-*-dev Application 자동 Sync
  → dev 환경 배포

[prod 수동 배포]
Jenkins (BRANCH=main)
  → docker push  <image>:prod-YYYYMMDD-HHmm
  → Image Updater (2분 폴링, regex ^prod-...)
    → gitops main 브랜치 values/prod/*-values.yaml 의 image.tag 갱신 (git commit & push)
  → ArgoCD insideinfo-*-prod Application 은 OutOfSync 상태가 됨
  → 담당자가 ArgoCD UI / CLI 에서 수동 Sync → prod 배포
```

### write-back 방식: git

Image Updater 는 gitops 레포에 직접 commit & push 한다.
- dev Application 은 `targetRevision: dev`, `syncPolicy.automated` 활성 → 자동 반영
- prod Application 은 `targetRevision: main`, `syncPolicy.automated` 미설정 → 수동 Sync 필요

> **참고**: ArgoCD v3 에서는 `argocd` write-back 방식(Application API 직접 수정)이 호환되지 않아 `git` 방식을 사용한다.

### 태그 네이밍 규칙

| 브랜치 | 이미지 태그 | Image Updater regex |
|--------|-------------|---------------------|
| `dev`  | `dev-YYYYMMDD-HHmm`  | `^dev-[0-9]{8}-[0-9]{4}$` |
| `main` | `prod-YYYYMMDD-HHmm` | `^prod-[0-9]{8}-[0-9]{4}$` |

Jenkinsfile 에서 `params.BRANCH` 에 따라 prefix 를 자동 부여한다 (Jenkinsfile-api / Jenkinsfile-web 참고).

## Step 1: ArgoCD Image Updater 설치 (또는 업그레이드)

### 신규 설치

```bash
helm repo add argo https://argoproj.github.io/argo-helm
helm repo update

helm upgrade --install argocd-image-updater argo/argocd-image-updater \
  --namespace argocd \
  --version 1.1.5 \
  --set config.logLevel=info \
  --wait --timeout 300s
```

### 기존 v0.x 에서 업그레이드

v0.x (annotation 기반) → v1.x (CRD 기반) 전환 시 `--reuse-values` 불가, 새로 설치한다.

```bash
helm upgrade --install argocd-image-updater argo/argocd-image-updater \
  --namespace argocd \
  --version 1.1.5 \
  --set config.logLevel=info \
  --wait --timeout 300s
```

### 설치 확인

```bash
kubectl get deploy -n argocd | grep image-updater
# argocd-image-updater-controller    1/1     1            1           ...
```

## Step 2: GitLab Registry 접근용 Secret 생성

Image Updater 가 GitLab Container Registry 에서 이미지 태그 목록을 조회하기 위한 인증 정보.

```bash
kubectl create secret docker-registry gitlab-registry-secret \
  -n argocd \
  --docker-server=https://gitlab.insideinfo.co.kr:5005 \
  --docker-username=<GitLab 사용자명> \
  --docker-password=<Access Token (read_registry scope)>
```

---

## Step 3: Registry 설정 (ConfigMap)

자체 서명 인증서 환경이므로 `insecure: true` 설정 필요.

```bash
kubectl edit configmap argocd-image-updater-config -n argocd
```

`data` 부분을 아래와 같이 수정:

```yaml
data:
  log.level: info
  registries.conf: |
    registries:
      - name: GitLab Registry
        prefix: gitlab.insideinfo.co.kr:5005
        api_url: https://gitlab.insideinfo.co.kr:5005
        credentials: pullsecret:argocd/gitlab-registry-secret
        insecure: true
```

### credentials 형식 참고

| 형식 | 문법 | Secret 타입 |
|------|------|-------------|
| `pullsecret:` | `pullsecret:namespace/name` | docker-registry 타입 |
| `secret:` | `secret:namespace/name#field` | generic 타입 (field 값은 `username:password` 형식) |

> **주의**: `secret:` 형식은 반드시 `#field` 가 필요하다. 누락 시 `invalid secret definition` 에러 발생.

수정 후 재시작:

```bash
kubectl rollout restart deployment argocd-image-updater-controller -n argocd
```

## Step 5: ArgoCD repo credential 에 write 권한 부여

Image Updater 는 ArgoCD 의 repo credential 을 사용해 gitops 레포에 push 한다.
해당 토큰은 **insideinfo-gitops 프로젝트**에서 발급한 것이어야 하며 `write_repository` scope 필수.

### ArgoCD repo credential 확인

```bash
kubectl get secret -n argocd -l argocd.argoproj.io/secret-type=repository -o yaml
```

### 토큰 업데이트 (필요 시)

GitLab > **insideinfo-gitops** > Settings > Access Tokens 에서 발급:
- Role: **Maintainer**
- Scopes: `read_repository`, `write_repository`

```bash
kubectl patch secret <repo-secret-name> -n argocd --type=merge \
  -p "{\"data\":{\"password\":\"$(echo -n '<새 토큰>' | base64)\"}}"
```

> **주의**: 다른 프로젝트(예: insideinfo-api) 토큰으로는 gitops 레포 push 권한이 없다.

## Step 6: ImageUpdater CR 생성 (dev + prod)

v1.x 는 Application annotation 대신 **ImageUpdater CRD** 를 사용한다.
dev / prod 각각 별도 CR 을 생성해 **같은 이미지 레포지토리에서 서로 다른 태그 prefix** 만 감지하게 한다.

### 6-1. insideinfo-api (dev)

```yaml
# apps/insideinfo-api/argocd/image-updater-dev.yaml
apiVersion: argocd-image-updater.argoproj.io/v1alpha1
kind: ImageUpdater
metadata:
  name: insideinfo-api-dev-updater
  namespace: argocd
spec:
  commonUpdateSettings:
    pullSecret: pullsecret:argocd/gitlab-registry-secret
    updateStrategy: newest-build
    forceUpdate: true
    allowTags: "regexp:^dev-[0-9]{8}-[0-9]{4}$"
  writeBackConfig:
    method: git
    gitConfig:
      branch: dev
      writeBackTarget: "helmvalues:./values/dev/inside-api-values.yaml"
  applicationRefs:
    - namePattern: "insideinfo-api-dev"
      images:
        - alias: api
          imageName: gitlab.insideinfo.co.kr:5005/ax-2/insideinfo-api/insideinfo-api
          manifestTargets:
            helm:
              name: image.repository
              tag: image.tag
```

## 트러블슈팅

### `skipping app of type '' because it's not of supported source type`

- **원인**: ArgoCD v3 에서 `argocd` write-back 방식 사용 시 발생
- **해결**: write-back 방식을 `git` 으로 변경

### `invalid secret definition: argocd/gitlab-registry-secret`

- **원인**: `secret:` 형식에서 `#field` 누락
- **해결**: `pullsecret:argocd/gitlab-registry-secret` 형식 사용 (docker-registry 타입 secret)

### `fatal: couldn't find remote ref dev`

- **원인**: gitops 레포에 dev 브랜치 미생성
- **해결**: `git checkout -b dev && git push origin dev`

### `You are not allowed to upload code` (403)

- **원인**: ArgoCD repo credential 토큰이 gitops 레포 write 권한 없음
- **해결**: insideinfo-gitops 프로젝트에서 Maintainer 역할 토큰 발급 후 ArgoCD secret 업데이트

### `"latest" strategy has been renamed to "newest-build"`

- **원인**: v1.x 에서 strategy 이름 변경
- **해결**: `updateStrategy: latest` → `updateStrategy: newest-build`

### prod 태그가 dev values 에 반영되거나 그 반대 현상

- **원인**: Jenkins 에서 prefix (`dev-` / `prod-`) 없이 push → 양쪽 regex 어느 쪽에도 매칭 안 되거나 오매칭
- **해결**: Jenkinsfile 의 `env.VERSION` 이 `dev-YYYYMMDD-HHmm` / `prod-YYYYMMDD-HHmm` 형식인지 확인

### prod 가 자동으로 배포됨

- **원인**: prod Application 에 `syncPolicy.automated` 가 설정되어 있음
- **해결**: `application-prod.yaml` 에서 `automated` 블록 제거 (02 가이드 Step 5 참고)

## 설정 요약

| 항목 | dev | prod |
|------|-----|------|
| Image Updater CR | `image-updater-dev.yaml` | `image-updater-prod.yaml` |
| allowTags regex | `^dev-[0-9]{8}-[0-9]{4}$` | `^prod-[0-9]{8}-[0-9]{4}$` |
| write-back 대상 브랜치 | `dev` | `main` |
| write-back target | `values/dev/*-values.yaml` | `values/prod/*-values.yaml` |
| ArgoCD App | `insideinfo-*-dev` (자동 Sync) | `insideinfo-*-prod` (수동 Sync) |
| Jenkins 이미지 태그 | `dev-YYYYMMDD-HHmm` | `prod-YYYYMMDD-HHmm` |

공통:
- Image Updater 버전: Helm chart 1.1.5 / App v1.1.1
- write-back 방식: git (ArgoCD v3 호환)
- 폴링 간격: 2분
- Registry 인증: `pullsecret:argocd/gitlab-registry-secret`
- Git 인증: ArgoCD repo credential (write 권한 토큰)
