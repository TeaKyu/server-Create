## 반영 완료

### ✅ `apps/insideinfo-web/helm/templates/ingress.yaml`

`sectionName` 조건부 지원 추가 — HTTPS 리스너에만 HTTPRoute가 연결되도록.

```yaml
  parentRefs:
  - name: {{ .Values.httproute.gatewayName }}
    namespace: {{ .Values.httproute.gatewayNamespace }}
    {{- if .Values.httproute.sectionName }}
    sectionName: {{ .Values.httproute.sectionName }}
    {{- end }}
```

### ✅ `apps/insideinfo-web/helm/templates/tls-secret.yaml` (신규)

TLS Secret + ReferenceGrant Helm 템플릿으로 관리.
- `Secret`: `insideinfo` 네임스페이스에 TLS 인증서 저장
- `ReferenceGrant`: `default` 네임스페이스 Gateway가 `insideinfo` 네임스페이스 Secret 참조 허용

### ✅ `apps/insideinfo-web/helm/values/prod/inside-web-values.yaml`

```yaml
httproute:
  sectionName: https
  hostnames:
  - prod.insideinfo
  - www.insideinfo.co.kr
  - insideinfo.co.kr
tls:
  enabled: true
  secretName: insideinfo-web-prod-tls
  crt: ""   # secret-values.enc.yaml 에서 주입
  key: ""   # secret-values.enc.yaml 에서 주입
```

### ✅ `apps/insideinfo-web/argocd/application-prod.yaml`

```yaml
      valueFiles:
        - values/prod/inside-web-values.yaml
        - secrets://values/prod/secret-values.enc.yaml
```

### ✅ ArgoCD `helm.valuesFileSchemes` 설정

`secrets://` 스킴 허용. 미설정 시 `URL scheme 'secrets' is not allowed` 에러 발생.

```bash
kubectl patch configmap argocd-cm -n argocd --type merge \
  -p '{"data":{"helm.valuesFileSchemes": "secrets,secrets+gpg-import,secrets+gpg-import-kubernetes,secrets+age-import,secrets+age-import-kubernetes,secrets+literal,https,http"}}'
kubectl rollout restart deployment argocd-repo-server -n argocd
```

> `k3s-apps.sh [8-2]`에 영구 반영 완료.

### ✅ NodePort 30443 추가

```bash
kubectl patch svc envoy-default-eg-e41e7b31 -n envoy-gateway-system --type merge \
  -p '{"spec":{"ports":[
    {"name":"http","port":80,"targetPort":80,"nodePort":30080,"protocol":"TCP"},
    {"name":"https","port":443,"targetPort":443,"nodePort":30443,"protocol":"TCP"}
  ]}}'
```

### ✅ Gateway HTTPS 리스너 추가

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: eg
  namespace: default
spec:
  gatewayClassName: eg
  listeners:
  - name: http
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: All
  - name: https
    protocol: HTTPS
    port: 443
    tls:
      mode: Terminate
      certificateRefs:
      - name: insideinfo-web-prod-tls
        namespace: insideinfo
    allowedRoutes:
      namespaces:
        from: All
EOF
```

### ✅ `apps/insideinfo-api/**` — 변경 없음

API는 내부 도메인(`prod.insideinfo.api`)만 사용하므로 TLS 적용 대상 아님.

## 트러블슈팅 기록

### SOPS 들여쓰기 오류 (2026-04-21)

**증상**: Gateway에서 `Secret must contain tls.crt and tls.key` 에러 → TLS Secret 내용이 비어있음.

**원인**: `secret-values.enc.yaml` 편집 시 인증서 내용 들여쓰기 4칸 미적용.

```yaml
# 잘못된 형태
tls:
  crt: |
-----BEGIN CERTIFICATE-----   ← 들여쓰기 없음

# 올바른 형태
tls:
  crt: |
    -----BEGIN CERTIFICATE-----   ← 4칸 들여쓰기
```

**조치**: 파일 삭제 후 `SOPS_AGE_KEY_FILE` 환경변수 명시하여 재생성.

---

### ArgoCD `secrets://` 스킴 차단 (2026-04-21)

**증상**: `error resolving value file path: the URL scheme 'secrets' is not allowed`

**원인**: ArgoCD 보안 설정에서 `secrets://` URL 스킴 기본 차단.

**조치**: `argocd-cm` ConfigMap에 `helm.valuesFileSchemes` 추가 후 repo-server 재시작.  
`k3s-apps.sh [8-2]`에 영구 반영.

## 작업 필요

### ① secret-values.enc.yaml 재생성 (인증서 내용 수정)

> 작업 위치: **로컬 PC**

```bash
cd /path/to/insideinfo-gitops

# 기존 파일 삭제 후 재생성
rm apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml
export SOPS_AGE_KEY_FILE=~/.config/sops/age/keys.txt
sops apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml
```

편집기에서 아래 형태로 입력 (들여쓰기 4칸 필수):

```yaml
tls:
  crt: |
    -----BEGIN CERTIFICATE-----
    (서버 인증서 내용)
    -----END CERTIFICATE-----
    -----BEGIN CERTIFICATE-----
    (체인 인증서 내용)
    -----END CERTIFICATE-----
  key: |
    -----BEGIN PRIVATE KEY-----
    (개인키 내용)
    -----END PRIVATE KEY-----
```

저장 후 push:

```bash
git add apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml
git commit -m "feat: prod TLS 인증서 secret 추가"
git push origin main
```

### ② 서버에서 application-prod.yaml 재적용

```bash
kubectl apply -f apps/insideinfo-web/argocd/application-prod.yaml
```

### ③ ArgoCD Hard Refresh → Sync

ArgoCD UI → `insideinfo-web-prod` → Refresh 드롭다운 → **Hard Refresh** → **Sync**

### ④ HTTP → HTTPS 리다이렉트 적용

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: http-to-https-redirect
  namespace: default
spec:
  parentRefs:
  - name: eg
    namespace: default
    sectionName: http
  hostnames:
  - www.insideinfo.co.kr
  - insideinfo.co.kr
  rules:
  - filters:
    - type: RequestRedirect
      requestRedirect:
        scheme: https
        statusCode: 301
EOF
```

### ⑤ 최종 확인

```bash
kubectl get secret insideinfo-web-prod-tls -n insideinfo
kubectl get referencegrant -n insideinfo
kubectl describe gateway eg | grep -A 5 "https"
curl -v https://www.insideinfo.co.kr
curl -I http://www.insideinfo.co.kr   # 301 확인
```
