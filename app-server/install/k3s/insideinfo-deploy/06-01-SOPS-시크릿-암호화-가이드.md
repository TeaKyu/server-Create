# SOPS 시크릿 암호화 가이드

기존 Sealed Secrets를 SOPS(Secrets OPerationS) + age로 교체하여 Git 레포의 시크릿을 암호화하는 가이드.

> **적용 완료**: 2026-04-16 dev 환경 적용 완료, 검증 완료.

---

## 배경

| 현재 상태 | 변경 |
|-----------|------|
| DB 비밀번호가 gitops values 파일에 평문 저장 | SOPS로 암호화하여 Git 커밋 |
| Sealed Secrets 설치됨 → 미사용 | SOPS로 대체, k3s-apps.sh에서 주석 처리됨 |

### 왜 SOPS인가?

| 비교 | Sealed Secrets | SOPS + age |
|------|----------------|------------|
| 서버 컴포넌트 | 컨트롤러 필요 (~64MB) | **불필요** (CLI만) |
| 암호화 위치 | 클러스터 내부에서만 암호화 가능 | 어디서든 암호화 가능 |
| 키 관리 | 컨트롤러에 종속 (컨트롤러 삭제 시 복호 불가) | age 키 파일만 보관하면 됨 |
| GitOps 궁합 | 좋음 | **최고** (helm-secrets 플러그인) |
| 멀티 클러스터 | 클러스터마다 다시 암호화 | 같은 키면 어디서든 복호 |
| CNCF | ✗ | ✓ (CNCF Sandbox) |

> 규모가 커져서 동적 시크릿, 감사로그 등이 필요하면 OpenBao로 전환 가능 (06-03 가이드 참고).

## 개념

```
[개발자 PC]                          [k3s 클러스터]

values.yaml (평문)                   ArgoCD repo-server
    │                                  ├ sops (바이너리)
    ▼                                  ├ helm-secrets (플러그인)
sops -e → values.enc.yaml ──push──▶   └ age 비밀키 (K8s Secret)
          (암호화된 상태로 Git 저장)          │
                                          ▼
                                  secrets:// 접두사 감지
                                  → sops -d 로 복호화
                                  → Helm values로 주입
                                          │
                                          ▼
                                     K8s Secret 생성
```

- **암호화**: 개발자 PC에서 `sops` CLI로 수행
- **복호화**: ArgoCD가 Helm 렌더링 시 helm-secrets 플러그인 + sops로 자동 수행
- **키**: age (현대적 암호화 도구, GPG보다 간단)

## Step 1: age 키 생성 (개발자 PC)

age는 SSH 키처럼 공개키/비밀키 쌍을 사용한다.

```bash
# age 설치
# macOS
brew install age

# Linux (Ubuntu/Debian)
sudo apt install age
```

```bash
# 키 생성
age-keygen -o age.key

# 출력 예시:
# Public key: age14mzxcffuanlxdwvvv47gcmn9kh6gy8d3yn3r6u3gya5ddt4xr3vqxc47kr
```

생성된 파일:
- `age.key` — **비밀키** (절대 Git에 커밋하지 않는다)
- 출력된 `Public key: age1...` — **공개키** (이것만 `.sops.yaml`에 기록)

> **중요**: `age.key` 파일을 안전한 곳에 백업한다. 이 키를 분실하면 암호화된 시크릿을 복호할 수 없다.

```bash
# age.key를 안전한 위치로 이동
mkdir -p ~/.config/sops/age
mv age.key ~/.config/sops/age/age.key
cp ~/.config/sops/age/age.key ~/.config/sops/age/keys.txt
```

## Step 2: SOPS 설치 (개발자 PC)

```bash
# macOS
brew install sops

# Linux
curl -LO https://github.com/getsops/sops/releases/download/v3.9.4/sops-v3.9.4.linux.amd64
chmod +x sops-v3.9.4.linux.amd64
sudo mv sops-v3.9.4.linux.amd64 /usr/local/bin/sops

# 설치 확인
sops --version
```

## Step 3: gitops 레포에 `.sops.yaml` 설정

gitops 레포 루트에 `.sops.yaml` 파일을 생성한다.

```yaml
creation_rules:
  - path_regex: secret-values(\.enc)?\.yaml$
    age: age14mzxcffuanlxdwvvv47gcmn9kh6gy8d3yn3r6u3gya5ddt4xr3vqxc47kr
```

> `path_regex`로 `secret-values.yaml` 또는 `secret-values.enc.yaml` 파일만 대상으로 지정.

## Step 4: 시크릿 values 파일 분리 및 암호화

현재 `inside-api-values.yaml`에 시크릿이 평문으로 포함되어 있다. 시크릿 부분을 별도 파일로 분리한 뒤 암호화한다.

### 4-1. 시크릿 전용 파일 생성 (임시, 암호화 전)

**dev 환경** — `apps/insideinfo-api/helm/values/dev/secret-values.yaml`:

```yaml
secret:
  data:
    config:
      application-secret.yml: |
        spring:
          datasource:
            url: jdbc:mysql://192.168.70.142:13306/INSIDEDB?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
            username: INSIDE
            password: 1qaz@WSX3edc
```

**prod 환경** — `apps/insideinfo-api/helm/values/prod/secret-values.yaml`:

```yaml
secret:
  data:
    config:
      application-secret.yml: |
        spring:
          datasource:
            url: jdbc:mysql://192.168.70.142:3306/INSIDEDB?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Seoul&characterEncoding=UTF-8
            username: INSIDE
            password: 1qaz@WSX3edc
```

### 4-2. SOPS로 암호화

```bash
export SOPS_AGE_KEY_FILE=~/.config/sops/age/age.key

# dev 환경 암호화
sops -e apps/insideinfo-api/helm/values/dev/secret-values.yaml \
  > apps/insideinfo-api/helm/values/dev/secret-values.enc.yaml

# prod 환경 암호화
sops -e apps/insideinfo-api/helm/values/prod/secret-values.yaml \
  > apps/insideinfo-api/helm/values/prod/secret-values.enc.yaml
```

### 4-3. 평문 파일 삭제 및 .gitignore 추가

```bash
rm apps/insideinfo-api/helm/values/dev/secret-values.yaml
rm apps/insideinfo-api/helm/values/prod/secret-values.yaml

echo "secret-values.yaml" >> .gitignore
echo "age.key" >> .gitignore
```

### 4-4. 기존 values 파일에서 시크릿 제거

`inside-api-values.yaml` (dev, prod 모두)에서 `secret:` 블록 전체를 삭제한다.

### 4-5. 최종 파일 구조

```
insideinfo-gitops/
├── .sops.yaml                              ← SOPS 설정 (공개키)
├── .gitignore                              ← secret-values.yaml, age.key 제외
├── apps/insideinfo-api/helm/
│   └── values/
│       ├── dev/
│       │   ├── inside-api-values.yaml      ← 시크릿 제거됨 (평문 설정만)
│       │   └── secret-values.enc.yaml      ← 암호화된 시크릿 (Git 커밋 OK)
│       └── prod/
│           ├── inside-api-values.yaml      ← 시크릿 제거됨
│           └── secret-values.enc.yaml      ← 암호화된 시크릿
```

## Step 5: ArgoCD에 SOPS + helm-secrets 설치

ArgoCD repo-server에 sops 바이너리와 helm-secrets 플러그인을 initContainer로 설치한다.

### 5-1. age 비밀키를 클러스터에 등록

개발자 PC에서 서버로 복사:

```bash
scp ~/.config/sops/age/age.key root@192.168.70.142:/tmp/age.key
```

k3s 마스터 노드에서 실행:

```bash
kubectl create secret generic sops-age \
  --namespace argocd \
  --from-file=keys.txt=/tmp/age.key

rm /tmp/age.key
```

확인:

```bash
kubectl get secret sops-age -n argocd
```

### 5-2. ArgoCD Helm 업그레이드

`k3s-apps.sh`의 [8] 섹션에 아래 values가 반영되어 있다. 수동 실행 시:

values 파일 생성 (vi로 작성하는 것을 권장):

`vi /tmp/argocd-values.yaml`:

```yaml
configs:
  params:
    server.insecure: true
repoServer:
  env:
    - name: SOPS_AGE_KEY_FILE
      value: /app/config/age/keys.txt
    - name: HELM_PLUGINS
      value: /custom-tools/helm-plugins
  volumes:
    - name: sops-age
      secret:
        secretName: sops-age
    - name: custom-tools
      emptyDir: {}
  volumeMounts:
    - name: sops-age
      mountPath: /app/config/age
    - name: custom-tools
      mountPath: /usr/local/bin/sops
      subPath: sops
    - name: custom-tools
      mountPath: /custom-tools/helm-plugins
      subPath: helm-plugins
  initContainers:
    - name: install-sops
      image: alpine:3.20
      command: ["sh","-c","wget -qO /custom-tools/sops https://github.com/getsops/sops/releases/download/v3.9.4/sops-v3.9.4.linux.amd64 && chmod +x /custom-tools/sops"]
      volumeMounts:
        - name: custom-tools
          mountPath: /custom-tools
    - name: install-helm-secrets
      image: alpine/helm:3.16.4
      command: ["sh","-c","helm plugin install https://github.com/jkroepke/helm-secrets --version v4.6.2 && printf 'name: secrets\\nversion: 4.6.2\\nusage: Secrets encryption in Helm\\ndescription: Secrets values encryption for Helm charts\\nuseTunnel: false\\ncommand: $HELM_PLUGIN_DIR/scripts/run.sh\\ndownloaders:\\n  - command: scripts/run.sh downloader\\n    protocols:\\n      - secrets\\n      - secrets+gpg-import\\n      - secrets+gpg-import-kubernetes\\n      - secrets+age-import\\n      - secrets+age-import-kubernetes\\n      - secrets+literal\\n' > /root/.local/share/helm/plugins/helm-secrets/plugin.yaml && cp -r /root/.local/share/helm/plugins/* /custom-tools/helm-plugins/"]
      volumeMounts:
        - name: custom-tools
          mountPath: /custom-tools/helm-plugins
          subPath: helm-plugins
```

> **주의**: helm-secrets v4.6.2의 `plugin.yaml`에 `platformCommand`(Windows용)와 `command`가 동시에 있으면
> Helm이 WARNING을 출력하고 ArgoCD JSON 파싱이 실패한다.
> initContainer에서 `plugin.yaml`을 `platformCommand` 없이 덮어써서 이 문제를 회피한다.

ArgoCD 업그레이드:

```bash
helm upgrade --install argocd argo/argo-cd \
  --namespace argocd \
  -f /tmp/argocd-values.yaml \
  --wait --timeout 600s
```

### 5-3. 설치 확인

```bash
# helm-secrets 플러그인 확인 (WARNING 없이 출력되어야 함)
kubectl exec -n argocd deploy/argocd-repo-server -- helm plugin list

# age 키 확인
kubectl exec -n argocd deploy/argocd-repo-server -- cat /app/config/age/keys.txt

# SOPS 바이너리 확인
kubectl exec -n argocd deploy/argocd-repo-server -- sops --version

# plugin.yaml에 platformCommand가 없는지 확인
kubectl exec -n argocd deploy/argocd-repo-server -- cat /custom-tools/helm-plugins/helm-secrets/plugin.yaml
```

## Step 6: ArgoCD Application 수정

ArgoCD Application에서 암호화된 values 파일을 `secrets://` 접두사로 참조한다.

`apps/insideinfo-api/argocd/application-dev.yaml`:

```yaml
spec:
  source:
    repoURL: https://gitlab.insideinfo.co.kr/ax-2/insideinfo-gitops.git
    targetRevision: dev
    path: apps/insideinfo-api/helm
    helm:
      valueFiles:
        - values/dev/inside-api-values.yaml
        - secrets://values/dev/secret-values.enc.yaml
```

`apps/insideinfo-api/argocd/application-prod.yaml`:

```yaml
spec:
  source:
    repoURL: https://gitlab.insideinfo.co.kr/ax-2/insideinfo-gitops.git
    targetRevision: main
    path: apps/insideinfo-api/helm
    helm:
      valueFiles:
        - values/prod/inside-api-values.yaml
        - secrets://values/prod/secret-values.enc.yaml
```

`secrets://` 접두사가 helm-secrets에게 복호화를 지시한다.

## Step 8: 시크릿 수정 방법 (일상 운영)

시크릿을 변경해야 할 때:

```bash
export SOPS_AGE_KEY_FILE=~/.config/sops/age/age.key

# 방법 1: 에디터로 열기 (자동 복호 → 편집 → 자동 재암호화)
sops apps/insideinfo-api/helm/values/dev/secret-values.enc.yaml

# 방법 2: 수동 복호 → 편집 → 수동 재암호화
sops -d secret-values.enc.yaml > secret-values.yaml
vi secret-values.yaml
sops -e secret-values.yaml > secret-values.enc.yaml
rm secret-values.yaml   # 평문 삭제 필수!
```

gitops 레포에 push하면 ArgoCD가 자동 반영.

## 트러블슈팅

### ArgoCD ComparisonError: Object 'Kind' is missing

```
error unmarshaling JSON: Object 'Kind' is missing in - {"WARNING":"both 'platformCommand' and 'command' ..."}
```

원인: helm-secrets의 `plugin.yaml`에 `platformCommand`(Windows용)와 `command`가 동시 존재.

조치: initContainer에서 `plugin.yaml`을 덮어쓰도록 설정 (Step 5-2의 values 참고).
확인: `kubectl exec -n argocd deploy/argocd-repo-server -- cat /custom-tools/helm-plugins/helm-secrets/plugin.yaml`
→ `platformCommand` 블록이 없어야 함.

### ArgoCD Sync 시 시크릿 복호화 실패

```
Error: secrets file not decryptable
```

원인: repo-server에 age 키가 없거나 경로가 잘못됨.

조치:

```bash
kubectl get secret sops-age -n argocd
kubectl exec -n argocd deploy/argocd-repo-server -- cat /app/config/age/keys.txt
kubectl exec -n argocd deploy/argocd-repo-server -- env | grep SOPS
```

### sops "no matching creation_rule"

`.sops.yaml` 파일이 없거나 `path_regex`가 파일명과 매치되지 않음.

조치: `.sops.yaml`이 gitops 레포 루트에 있는지, 파일명이 `secret-values`로 시작하는지 확인.

### age 키 분실

age 비밀키를 분실하면 암호화된 시크릿을 복호할 수 없다.

조치:
1. DB에서 직접 비밀번호를 확인하거나 DBA에게 요청
2. 새 age 키 생성 후 시크릿을 다시 암호화
3. ArgoCD의 sops-age Secret 업데이트

**예방**: age.key를 팀 내 안전한 곳(비밀번호 관리자, 금고 등)에 백업.

### ArgoCD 캐시된 에러가 남아있을 때

repo-server 파드를 재시작하면 캐시가 초기화된다:

```bash
kubectl delete pod -n argocd -l app.kubernetes.io/component=repo-server
```

파드가 `1/1 Running`이 된 후 ArgoCD UI에서 **Hard Refresh**.
