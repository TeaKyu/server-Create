## 1. TLS 인증서 관리

### 현재 인증서 정보

| 항목 | 값 |
|------|-----|
| 발급기관 | GlobalSign GCC R6 AlphaSSL CA 2025 |
| 도메인 | `www.insideinfo.co.kr`, `insideinfo.co.kr` |
| 만료일 | **2027-02-15** |
| 저장 위치 | K8s Secret `insideinfo-web-prod-tls` (`insideinfo` 네임스페이스) |
| gitops 관리 | `apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml` (SOPS 암호화) |

### 인증서 만료일 확인

```bash
# 클러스터에서 만료일 확인
kubectl get secret insideinfo-web-prod-tls -n insideinfo \
  -o jsonpath='{.data.tls\.crt}' | base64 -d | openssl x509 -noout -dates

# 출력 예시
# notBefore=Jan 14 00:59:50 2026 GMT
# notAfter=Feb 15 00:59:49 2027 GMT
```

> **권장**: 만료 30일 전에 갱신 시작. 캘린더에 **2027-01-15** 알림 등록 권장.

## 2. 인증서 변경 절차

### 2-1. 신규 인증서 파일 준비

발급기관에서 아래 파일 수령:

| 파일 | 설명 |
|------|------|
| `cert.crt` | 서버 인증서 |
| `chain.crt` | 체인(중간) 인증서 |
| `cert.key` | 개인 키 |

체인 포함 fullchain 파일 생성:

```bash
cat cert.crt chain.crt > fullchain.crt

# 인증서 정보 확인
openssl x509 -in fullchain.crt -noout -subject -issuer -dates
```

### 2-2. secret-values.enc.yaml 업데이트

> 작업 위치: **로컬 PC** (SOPS age 개인키 필요)

```bash
cd /path/to/insideinfo-gitops

# SOPS로 파일 편집 (자동 복호화 → 편집 → 자동 암호화)
export SOPS_AGE_KEY_FILE=~/.config/sops/age/keys.txt
sops apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml
```

편집기에서 `tls.crt`, `tls.key` 값을 신규 인증서 내용으로 교체:

```yaml
tls:
  crt: |
    -----BEGIN CERTIFICATE-----
    (신규 fullchain.crt 내용 — 서버 인증서 + 체인 인증서)
    -----END CERTIFICATE-----
    -----BEGIN CERTIFICATE-----
    (체인 인증서 내용)
    -----END CERTIFICATE-----
  key: |
    -----BEGIN PRIVATE KEY-----
    (신규 cert.key 내용)
    -----END PRIVATE KEY-----
```

> **주의**: `crt:` / `key:` 아래 내용은 반드시 **4칸 들여쓰기** 적용.  
> `Bag Attributes`, `subject=`, `issuer=` 줄은 제외하고 `-----BEGIN` 부터만 입력.

저장 후 push:

```bash
git add apps/insideinfo-web/helm/values/prod/secret-values.enc.yaml
git commit -m "cert: TLS 인증서 갱신 (만료: YYYY-MM-DD)"
git push origin main
```

### 2-3. ArgoCD Sync

```bash
# 서버에서 application-prod.yaml 재적용 (신규 변경사항 있을 경우)
kubectl apply -f apps/insideinfo-web/argocd/application-prod.yaml

# ArgoCD CLI로 Sync
argocd app sync insideinfo-web-prod
```

또는 ArgoCD UI → `insideinfo-web-prod` → **Hard Refresh** → **Sync**

### 2-4. 인증서 적용 확인

```bash
# Secret 업데이트 시각 확인
kubectl get secret insideinfo-web-prod-tls -n insideinfo

# 실제 인증서 만료일 확인
kubectl get secret insideinfo-web-prod-tls -n insideinfo \
  -o jsonpath='{.data.tls\.crt}' | base64 -d | openssl x509 -noout -dates

# 브라우저에서 자물쇠 아이콘 → 인증서 정보 → 만료일 확인
```

## 3. 배포 운영

### prod 배포 절차

prod는 **수동 Sync** (자동 배포 없음). Jenkins 빌드 후 아래 순서로 진행:

```
Jenkins 빌드 완료 (prod-YYYYMMDD-HHmm 태그 push)
       ↓
ArgoCD Image Updater가 gitops main 브랜치 values 자동 업데이트
       ↓
ArgoCD UI에서 insideinfo-api-prod / insideinfo-web-prod OutOfSync 확인
       ↓
담당자 검토 후 수동 Sync
```

```bash
# ArgoCD CLI로 Sync
argocd app sync insideinfo-api-prod
argocd app sync insideinfo-web-prod
```

### 롤백 절차

ArgoCD UI → 해당 App → **HISTORY AND ROLLBACK** → 이전 버전 선택 → **Rollback**

또는 gitops에서 이전 태그로 values 되돌리기:

```bash
# values 파일에서 image.tag를 이전 버전으로 수정
vi apps/insideinfo-web/helm/values/prod/inside-web-values.yaml
# image.tag: "prod-20260415-1456"  ← 이전 버전으로

git add .
git commit -m "fix: prod 롤백 (prod-20260415-1456)"
git push origin main
# ArgoCD에서 수동 Sync
```

## 4. 장애 대응

### HTTPS 접속 불가

```bash
# 1. 인증서 Secret 존재 여부
kubectl get secret insideinfo-web-prod-tls -n insideinfo

# 2. Gateway HTTPS 리스너 상태
kubectl describe gateway eg -n default | grep -A 30 "Status"

# 3. ReferenceGrant 존재 여부
kubectl get referencegrant -n insideinfo

# 4. NodePort 30443 확인
kubectl get svc -n envoy-gateway-system | grep 30443

# 5. 파드 상태
kubectl get pods -n insideinfo
```

### ArgoCD Sync 실패

```bash
# 에러 확인
kubectl logs -n argocd deploy/argocd-repo-server --tail=50 | grep -i error

# SOPS 복호화 실패 시 → age 키 Secret 확인
kubectl get secret sops-age -n argocd

# secrets:// 스킴 차단 시 → ConfigMap 확인
kubectl get configmap argocd-cm -n argocd -o yaml | grep valuesFileSchemes
# 없으면 아래 명령 실행:
kubectl patch configmap argocd-cm -n argocd --type merge \
  -p '{"data":{"helm.valuesFileSchemes": "secrets,secrets+gpg-import,secrets+gpg-import-kubernetes,secrets+age-import,secrets+age-import-kubernetes,secrets+literal,https,http"}}'
kubectl rollout restart deployment argocd-repo-server -n argocd
```

### 파드 재시작 반복 (CrashLoopBackOff)

```bash
# 로그 확인
kubectl logs -n insideinfo -l app.kubernetes.io/name=insideinfo-web --tail=100
kubectl logs -n insideinfo -l app.kubernetes.io/name=insideinfo-api --tail=100

# 이전 파드 로그 (재시작된 경우)
kubectl logs -n insideinfo <pod-name> --previous
```
