# 04. 서비스 접속 가이드

## 접속 방식 요약

| 환경 | 접속 흐름 | URL 형식 |
|------|-----------|----------|
| NAT 미사용 | 브라우저 → hosts → MetalLB IP → Envoy → Pod | `http://도메인` |
| NAT 사용, 외부 IP 확인 | 브라우저 → hosts → External IP → 공유기 포트포워딩 → ServiceLB → Envoy → Pod | `http(s)://도메인` |
| NAT 사용, 외부 IP 미확인 | 브라우저 → hosts → Private IP → ServiceLB → Envoy → Pod | `http://도메인` |

> NAT 사용 환경은 ServiceLB(k3s 내장)가 표준 포트(80/443)를 직접 바인딩한다.
> 외부 접속은 공유기에서 외부 80/443 → 서버 80/443 포트포워딩 설정이 필요하다.

---

## 1. hosts 파일 설정

### NAT 미사용

```bash
<GATEWAY_IP>  grafana.local
<GATEWAY_IP>  prometheus.local
<GATEWAY_IP>  argocd.local
<GATEWAY_IP>  dashboard.local
```

`GATEWAY_IP` 확인:

```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
```

### NAT 사용, 외부 IP 확인

```bash
<EXTERNAL_IP>  grafana.local
<EXTERNAL_IP>  prometheus.local
<EXTERNAL_IP>  argocd.local
<EXTERNAL_IP>  dashboard.local
```

### NAT 사용, 외부 IP 미확인

내부 테스트용이라면 private IP를 hosts에 매핑해서 도메인으로 접속하면 된다.

```bash
<PRIVATE_IP>  grafana.local
<PRIVATE_IP>  prometheus.local
<PRIVATE_IP>  argocd.local
<PRIVATE_IP>  dashboard.local
```

## 2. 서비스별 URL

### NAT 미사용

| 서비스        | URL                       | 인증                  |
| ---------- | ------------------------- | ------------------- |
| Grafana    | `http://grafana.local`    | `admin / admin1234` |
| Prometheus | `http://prometheus.local` | 없음                  |
| Argo CD    | `http://argocd.local`     | `admin / admin`     |
| Headlamp   | `http://dashboard.local`  | Bearer Token        |

### NAT 사용

| 서비스        | URL                       | 인증                  |
| ---------- | ------------------------- | ------------------- |
| Grafana    | `http://grafana.local`    | `admin / admin1234` |
| Prometheus | `http://prometheus.local` | 없음                  |
| Argo CD    | `http://argocd.local`     | `admin / admin`     |
| Headlamp   | `http://dashboard.local`  | Bearer Token        |

> ServiceLB가 표준 포트(80)를 직접 바인딩하므로 포트 없이 도메인으로 접속한다.
> Host 기반 라우팅이라서 IP만 직접 입력하면 원하는 서비스로 안 갈 수 있다.

## 4. 외부 kubectl 접속

이건 NAT 사용 환경에서 external IP를 이미 알고 있을 때만 의미가 있다.

```bash
scp root@<EXTERNAL_IP>:/etc/rancher/k3s/k3s.yaml ~/.kube/config
sed -i "s/127.0.0.1/<EXTERNAL_IP>/g" ~/.kube/config
kubectl get nodes
```

주의:

- `server:` 값은 `https://<EXTERNAL_IP>:6443` 형태여야 함
- `6443/tcp`가 외부에서 열려 있어야 함
- external IP를 나중에 반영했다면 `--tls-san EXTERNAL_IP`가 포함된 상태여야 함

## 5. 빠른 점검

### 서비스 공통

```bash
kubectl get nodes -o wide
kubectl get httproute
kubectl get pods -A | grep -v Running | grep -v Completed
```

### NAT 미사용

```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
kubectl get pods -n metallb-system
```

### NAT 사용

```bash
# ServiceLB 파드 상태 확인
kubectl get pods -n kube-system | grep svclb

# Envoy 서비스 External IP 할당 확인
kubectl get svc -n envoy-gateway-system

# 80/443 포트 바인딩 확인
ss -tlnp | grep -E '":80|":443'
```

## 6. 내 앱 추가하기

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app
  namespace: default
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-svc
      port: 3000
```

적용:

```bash
kubectl apply -f my-app-route.yaml
```

hosts 예시:

### NAT 미사용

```bash
echo "$(kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}')  my-app.local" | sudo tee -a /etc/hosts
```

### NAT 사용

```bash
TARGET_IP=$(grep EXTERNAL_IP /etc/k3s-env | cut -d= -f2)
[ -n "$TARGET_IP" ] || TARGET_IP=$(grep PRIVATE_IP /etc/k3s-env | cut -d= -f2)
echo "$TARGET_IP  my-app.local" | sudo tee -a /etc/hosts
```

접속:

- NAT 미사용: `http://my-app.local`
- NAT 사용: `http://my-app.local` (공유기 포트포워딩 설정 후)
