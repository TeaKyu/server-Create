# 04. K3s 서비스 접속 가이드

## 접속 방식: HTTPRoute 도메인 라우팅

모든 서비스는 Envoy Gateway의 HTTPRoute를 통해 도메인으로 접속합니다.
port-forward는 필요 없으며, 서버 재시작 후에도 유지됩니다.

```
브라우저 → hosts 파일 (도메인 → IP) → MetalLB IP → Envoy Gateway → HTTPRoute → Service → Pod
```

---

## 1. 사전 설정: 접속 PC의 hosts 파일

### Gateway IP 확인 (마스터에서)

```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
# 예: 192.168.1.200
```

### hosts 파일 수정

접속할 PC(브라우저를 실행하는 PC)의 hosts 파일에 추가합니다.

**Linux/Mac** — `/etc/hosts`

```bash
sudo bash -c 'cat >> /etc/hosts << EOF
192.168.1.200  grafana.local
192.168.1.200  prometheus.local
192.168.1.200  argocd.local
192.168.1.200  dashboard.local
EOF'
```

**Windows** — `C:\Windows\System32\drivers\etc\hosts`

메모장을 관리자 권한으로 열고 추가:

```
192.168.1.200  grafana.local
192.168.1.200  prometheus.local
192.168.1.200  argocd.local
192.168.1.200  dashboard.local
```

> IP는 위에서 확인한 Gateway IP로 변경하세요.

## 2. 서비스별 접속 정보

| 서비스 | URL | 인증 |
|--------|-----|------|
| Grafana | http://grafana.local | admin / admin1234 |
| Prometheus | http://prometheus.local | 없음 |
| Argo CD | http://argocd.local | admin / (아래 명령어로 확인) |
| Headlamp | http://dashboard.local | Bearer Token (아래 명령어로 생성) |

### ArgoCD 초기 비밀번호

```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d; echo
```

### ArgoCD 비밀번호 강제 초기화 (admin 으로)

```bash
NEW_HASH=$(kubectl -n argocd exec deploy/argocd-server -- argocd account bcrypt --password 'admin')
kubectl -n argocd patch secret argocd-secret \
  -p "{\"stringData\": {\"admin.password\": \"$NEW_HASH\", \"admin.passwordMtime\": \"$(date +%FT%T%Z)\"}}"
```

### Headlamp 로그인 토큰

```bash
kubectl -n headlamp create token headlamp
```

## 3. Grafana 설정

### 기본 대시보드

접속 후 좌측 메뉴 > Dashboards에 이미 Kubernetes 대시보드가 포함되어 있습니다:

- Kubernetes / Compute Resources / Cluster
- Kubernetes / Compute Resources / Namespace (Pods)
- Node Exporter / Nodes

### Loki 데이터소스 추가 (로그 조회)

1. 좌측 메뉴 > Connections > Data Sources
2. Add data source > Loki 선택
3. URL: `http://loki-gateway.logging.svc.cluster.local`
4. Save & Test

### 로그 조회

좌측 메뉴 > Explore > 상단 데이터소스를 "Loki"로 변경

```
{namespace="default"}                     # 전체 로그
{namespace="argocd", app="argocd-server"} # ArgoCD 로그
{pod="my-app-xxxxx"} |= "error"           # 특정 Pod에서 error 검색
```

# 1. 노드 상태

## 5. 내 앱에 HTTPRoute 추가하기

### 5-1. 앱의 Service 확인

```bash
kubectl get svc -n default
```

### 5-2. HTTPRoute 생성

```yaml
# my-app-route.yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app
  namespace: default
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-svc
      port: 3000
```

```bash
kubectl apply -f my-app-route.yaml
```

### 5-3. hosts 파일에 도메인 추가

```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
# 확인한 IP를 hosts 파일에 추가
# 192.168.1.200  my-app.local
```

### 경로 기반 라우팅 예시

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app-routes
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /api
    backendRefs:
    - name: api-svc
      port: 8080
  - matches:
    - path:
        type: PathPrefix
        value: /
    backendRefs:
    - name: frontend-svc
      port: 3000
```

### 트래픽 분할 예시 (Canary)

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app-canary
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-v1
      port: 80
      weight: 90
    - name: my-app-v2
      port: 80
      weight: 10
```
