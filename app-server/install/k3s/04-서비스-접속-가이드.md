# 04. K3s 서비스 접속 가이드

## 접속 방식 요약

| 환경 | 접속 흐름 | URL 형식 |
|------|---------|---------|
| 일반 서버 | 브라우저 → hosts (도메인→MetalLB IP) → Envoy → Pod | `http://도메인` |
| OCI | 브라우저 → hosts (도메인→Public IP) → OCI NAT → :30080 → Envoy → Pod | `http://도메인:30080` |

---

## 1. 사전 설정: 접속 PC의 hosts 파일

### 접속 IP 확인

**일반 서버** — MetalLB가 할당한 Gateway IP 확인 (서버에서 실행)
```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
# 예: 192.168.1.200
```

**OCI** — Public IP 확인 (서버에서 실행)
```bash
grep PUBLIC_IP /etc/k3s-env
# 예: PUBLIC_IP=150.xxx.xxx.xxx
```

또는 k3s-apps.sh 완료 시 화면에 출력된 IP를 그대로 사용합니다.

### hosts 파일 수정

**Linux/Mac** — `/etc/hosts`

```bash
# 일반 서버: GATEWAY_IP = MetalLB External IP
sudo bash -c 'cat >> /etc/hosts << EOF
<GATEWAY_IP>  grafana.local
<GATEWAY_IP>  prometheus.local
<GATEWAY_IP>  argocd.local
<GATEWAY_IP>  dashboard.local
EOF'

# OCI: OCI_PUBLIC_IP = Public IP 주소
sudo bash -c 'cat >> /etc/hosts << EOF
<OCI_PUBLIC_IP>  grafana.local
<OCI_PUBLIC_IP>  prometheus.local
<OCI_PUBLIC_IP>  argocd.local
<OCI_PUBLIC_IP>  dashboard.local
EOF'
```

**Windows** — `C:\Windows\System32\drivers\etc\hosts` (관리자 권한 메모장)

```
# 일반 서버 (MetalLB IP 사용)
192.168.1.200  grafana.local
192.168.1.200  prometheus.local
192.168.1.200  argocd.local
192.168.1.200  dashboard.local

# OCI (Public IP 사용)
150.xxx.xxx.xxx  grafana.local
150.xxx.xxx.xxx  prometheus.local
150.xxx.xxx.xxx  argocd.local
150.xxx.xxx.xxx  dashboard.local
```

## 2. 서비스별 접속 정보

### 일반 서버

| 서비스        | URL                     | 인증                        |
| ---------- | ----------------------- | ------------------------- |
| Grafana    | http://grafana.local    | admin / admin1234         |
| Prometheus | http://prometheus.local | 없음                        |
| Argo CD    | http://argocd.local     | admin / (아래 명령어로 확인)      |
| Headlamp   | http://dashboard.local  | Bearer Token (아래 명령어로 생성) |

### OCI (NodePort 30080 필수)

| 서비스        | URL                           | 인증                        |
| ---------- | ----------------------------- | ------------------------- |
| Grafana    | http://grafana.local:30080    | admin / admin             |
| Prometheus | http://prometheus.local:30080 | 없음                        |
| Argo CD    | http://argocd.local:30080     | admin / (아래 명령어로 확인)      |
| Headlamp   | http://dashboard.local:30080  | Bearer Token (아래 명령어로 생성) |

> **OCI에서 `:30080`을 빠뜨리면 접속이 되지 않습니다.**
> OCI Public IP로 들어온 요청이 NodePort 30080 → Envoy Gateway로 라우팅됩니다.

### ArgoCD 초기 비밀번호 확인

```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d; echo
```

### ArgoCD 비밀번호 강제 초기화

```bash
NEW_HASH=$(kubectl -n argocd exec deploy/argocd-server -- argocd account bcrypt --password 'admin')
kubectl -n argocd patch secret argocd-secret \
  -p "{\"stringData\": {\"admin.password\": \"$NEW_HASH\", \"admin.passwordMtime\": \"$(date +%FT%T%Z)\"}}"
```

### Headlamp 로그인 토큰

```bash
kubectl -n headlamp create token headlamp
```

## 3. [OCI 전용] 로컬 PC에서 kubectl 원격 접속

서버에 매번 SSH 접속하지 않고, **로컬 PC에서 kubectl을 직접 사용**하려면 kubeconfig를 가져와야 합니다.

### 설정 방법

**서버에서 Public IP 기반 kubeconfig 추출:**
```bash
# 서버에서 실행 — Private IP를 Public IP로 교체해서 출력
PUBLIC_IP=$(grep PUBLIC_IP /etc/k3s-env | cut -d= -f2)
sudo cat /etc/rancher/k3s/k3s.yaml | sed "s/127.0.0.1/$PUBLIC_IP/g"
```

**로컬 PC에서 설정 (방법 1 — SCP):**
```bash
# 로컬 PC에서 실행
mkdir -p ~/.kube
scp -i ~/.ssh/oci_key opc@<PUBLIC_IP>:/etc/rancher/k3s/k3s.yaml ~/.kube/config
sed -i "s/127.0.0.1/<PUBLIC_IP>/g" ~/.kube/config

# 접속 확인
kubectl get nodes
```

**로컬 PC에서 설정 (방법 2 — 복붙):**
```bash
# 서버에서 출력한 내용을 로컬 ~/.kube/config에 붙여넣기
vi ~/.kube/config
```

> OCI Security List에서 **6443/TCP**가 열려 있어야 원격 kubectl이 작동합니다.
> server 주소가 `https://<PUBLIC_IP>:6443` 형태여야 합니다.

### 주의사항

- kubeconfig의 `server:` 값이 반드시 **Public IP**여야 합니다 (Private IP X)
- TLS 인증서에 Public IP가 SAN으로 포함되어 있어서 인증서 오류 없이 접속됩니다
  (`--tls-san PUBLIC_IP`가 k3s-master.sh에서 설정됨)

## 4. Grafana 설정

### 기본 대시보드

접속 후 좌측 메뉴 > Dashboards에 이미 Kubernetes 대시보드가 포함되어 있습니다.
- Kubernetes / Compute Resources / Cluster
- Kubernetes / Compute Resources / Namespace (Pods)
- Node Exporter / Nodes

### Loki 데이터소스 추가 (로그 조회)

1. 좌측 메뉴 > Connections > Data Sources
2. Add data source > Loki 선택
3. URL: `http://loki-gateway.logging.svc.cluster.local`
4. Save & Test

### 로그 조회 (LogQL)

좌측 메뉴 > Explore > 상단 데이터소스를 "Loki"로 변경

```
{namespace="default"}                     # 전체 로그
{namespace="argocd", app="argocd-server"} # ArgoCD 로그
{pod="my-app-xxxxx"} |= "error"           # 특정 Pod에서 error 검색
```

## 5. 접속 안 될 때 빠른 체크리스트

```bash
# ─── 공통 ────────────────────────────────────────────────────

# 1. 노드 Ready 상태 확인
kubectl get nodes -o wide

# 2. HTTPRoute 목록 확인
kubectl get httproute

# 3. Pod 상태 확인
kubectl get pods -A | grep -v Running | grep -v Completed

# 4. K3s 서비스 상태
systemctl status k3s

# ─── 일반 서버 ───────────────────────────────────────────────

# 5. MetalLB Gateway External IP 확인
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
# IP가 없으면 MetalLB 문제
kubectl get pods -n metallb-system

# ─── OCI 전용 ────────────────────────────────────────────────

# 5. Envoy Gateway NodePort 30080 확인
kubectl get svc -n envoy-gateway-system
# TYPE=NodePort, PORT(S)에 30080이 있어야 함

# 6. iptables REJECT 룰 확인
sudo iptables -L INPUT -n | grep REJECT
# 있으면: sudo iptables -P INPUT ACCEPT && sudo iptables -F INPUT

# 7. OCI Security List 포트 확인 (브라우저로 OCI 콘솔 접속)
#    네트워킹 → VCN → 서브넷 → Security List → 수신 규칙
#    30080/TCP, 6443/TCP가 열려 있어야 함

# 8. 포트 실제 리스닝 확인
ss -tlnp | grep 30080
```

## 6. 내 앱에 HTTPRoute 추가하기

### 기본 HTTPRoute

```yaml
# my-app-route.yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app
  namespace: default
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-svc
      port: 3000
```

```bash
kubectl apply -f my-app-route.yaml
```

### hosts 파일에 도메인 추가

```bash
# 일반 서버
echo "$(kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}')  my-app.local" | sudo tee -a /etc/hosts

# OCI
echo "$(grep PUBLIC_IP /etc/k3s-env | cut -d= -f2)  my-app.local" | sudo tee -a /etc/hosts
```

접속:
- 일반 서버: `http://my-app.local`
- OCI: `http://my-app.local:30080`

### 경로 기반 라우팅

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app-routes
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /api
    backendRefs:
    - name: api-svc
      port: 8080
  - matches:
    - path:
        type: PathPrefix
        value: /
    backendRefs:
    - name: frontend-svc
      port: 3000
```

### 트래픽 분할 (Canary)

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app-canary
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-v1
      port: 80
      weight: 90
    - name: my-app-v2
      port: 80
      weight: 10
```
