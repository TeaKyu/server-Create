# 06. 워커 노드 확장 가이드

마스터 1대 운영 중인 클러스터에 워커를 추가하는 절차야.

---

## 기본 원칙

- 워커 조인은 항상 마스터의 실제 통신 가능한 IP로 한다
- NAT 미사용이면 `MASTER_DIRECT_IP`
- NAT 사용이면 `MASTER_PRIVATE_IP`
- NAT 사용 환경에서도 워커 external IP는 선택값이다

---

## 추가 포트

| 포트 | 프로토콜 | 용도 | 방향 |
|------|---------|------|------|
| 6443 | TCP | K3s API Server | 워커 → 마스터 |
| 10250 | TCP | kubelet | 마스터 ↔ 워커 |
| 8472 | UDP | Flannel VXLAN | 노드 간 양방향 |

---

## 1단계: 워커에서 공통 설정 실행

각 워커에 `k3s-common.sh`를 복사한 뒤 환경에 맞게 수정한다.

### NAT 미사용

```bash
USE_NAT=false
DIRECT_BIND_IP="<워커의 직접 바인딩 IP>"
MASTER_HOSTNAME="k3s-master"
```

### NAT 사용, 외부 IP 미확인

```bash
USE_NAT=true
EXTERNAL_IP=""
MASTER_HOSTNAME="k3s-master"
```

### NAT 사용, 외부 IP 확인

```bash
USE_NAT=true
EXTERNAL_IP="<워커의 external IP>"
MASTER_HOSTNAME="k3s-master"
```

실행:

```bash
sudo bash /opt/k3s-lab/k3s-common.sh
```

---

## 2단계: 마스터에서 토큰 확인

```bash
cat /var/lib/rancher/k3s/server/node-token
```

---

## 3단계: k3s-worker.sh 설정

### NAT 미사용

```bash
K3S_TOKEN="<마스터에서 복사한 토큰>"
MASTER_DIRECT_IP="<마스터의 직접 바인딩 IP>"
```

### NAT 사용

```bash
K3S_TOKEN="<마스터에서 복사한 토큰>"
MASTER_PRIVATE_IP="<마스터의 private IP>"
```

> NAT 사용 환경에서 워커는 마스터의 external IP가 아니라 private IP로 조인한다.

---

## 4단계: 워커 조인

```bash
sudo bash /opt/k3s-lab/k3s-worker.sh
```

확인:

```bash
kubectl get nodes -o wide
```

해석:

- NAT 사용 + external IP 미확인: `INTERNAL-IP`만 보여도 정상
- NAT 사용 + external IP 확인: `EXTERNAL-IP`까지 보이면 정상

---

## 5단계: 워커 추가 후 확인

### Pod 분산

```bash
kubectl get pods -A -o wide | grep -v Completed
kubectl rollout restart deployment <이름> -n <네임스페이스>
```

### Alloy DaemonSet

```bash
kubectl get pods -n logging -l app.kubernetes.io/name=alloy -o wide
```

---

## 워커 제거

마스터에서:

```bash
kubectl drain k3s-worker1 --ignore-daemonsets --delete-emptydir-data
kubectl delete node k3s-worker1
```

워커에서:

```bash
sudo bash /opt/k3s-lab/k3s-cleanup.sh
```

> `k3s-cleanup.sh`는 `k3s-agent-uninstall.sh` 실행 후 로컬 설정까지 같이 정리한다.

---

## 트러블슈팅

### 토큰 오류

```bash
cat /var/lib/rancher/k3s/server/node-token
/usr/local/bin/k3s-agent-uninstall.sh
sudo bash k3s-worker.sh
```

### API 연결 확인

NAT 미사용:

```bash
curl -k https://<MASTER_DIRECT_IP>:6443
```

NAT 사용:

```bash
curl -k https://<MASTER_PRIVATE_IP>:6443
```

### 로그 확인

```bash
journalctl -u k3s-agent -f --no-pager
```

### hosts 확인

```bash
ping k3s-master
grep k3s-master /etc/hosts
```

## 마스터에서 워크로드 실행 방지

```bash
kubectl taint nodes k3s-master node-role.kubernetes.io/master:NoSchedule
kubectl taint nodes k3s-master node-role.kubernetes.io/master:NoSchedule-
```
