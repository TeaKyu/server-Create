# 03. K3s 설치 단계별 가이드

## 전체 설치 흐름

```
[1단계] k3s-common.sh     → OS 설정 (3~5분)
[2단계] k3s-master.sh     → K3s Server 설치 (1~2분)
[3단계] k3s-apps.sh       → 전체 DevOps 스택 설치 (10~15분)
[4단계] 확인               → kubectl get nodes + Pod 상태
```

총 소요 시간: 약 15~25분 (인터넷 속도에 따라 다름)

---

## 1단계 — OS 기본 설정

마스터 서버에 root로 접속하여 실행합니다.

```bash
cd /opt/k3s-lab
chmod +x k3s-*.sh
sudo bash k3s-common.sh
# "공통 설정 완료!" 출력 확인
```

### k3s-common.sh가 하는 일

| 단계 | 내용 | 설명 |
|------|------|------|
| [0-1] | root 비밀번호 | `root:1234` |
| [0-2] | 타임존 | Asia/Seoul, NTP 동기화 |
| [0-3] | 기본 패키지 | dnf-utils, iproute-tc, git, curl |
| [0-4] | /etc/hosts | 마스터 IP-호스트명 매핑 |
| [0-5] | 방화벽 해제 | firewalld 중지/비활성화 |
| [0-6] | Swap 비활성화 | kubelet 요구사항 |
| [0-7] | SELinux | Permissive 모드 |
| [1] | iptables | overlay, br_netfilter 모듈 + sysctl |
| [2] | 한글 | glibc-langpack-ko |

> K8s와 달리 containerd, kubeadm 설치 단계가 없습니다.
> K3s가 containerd를 자동으로 포함합니다.

---

## 2단계 — K3s Server 설치

```bash
sudo bash k3s-master.sh
```

### k3s-master.sh가 하는 일

| 단계 | 내용 | 설명 |
|------|------|------|
| [1] | K3s Server 설치 | Traefik/ServiceLB 비활성화 포함 |
| [2] | kubectl 설정 | kubeconfig 복사, KUBECONFIG 환경변수 |
| [3] | kubectl 편의기능 | bash 자동완성, alias k=kubectl |
| [4] | 노드 상태 확인 | `kubectl get nodes` |

### 완료 확인

```bash
kubectl get nodes
# NAME          STATUS   ROLES                  AGE   VERSION
# k3s-master    Ready    control-plane,master   1m    v1.32.x+k3s1
```

> K3s는 설치 즉시 Ready 상태가 됩니다. Flannel이 내장되어 있어 별도 CNI 대기가 없습니다.

## 3단계 — 전체 DevOps 스택 설치

```bash
sudo bash k3s-apps.sh
```

### k3s-apps.sh가 하는 일 (순서대로)

| 단계 | 컴포넌트 | 설치 방식 | 소요 시간 |
|------|---------|----------|----------|
| [1] | Helm 4.x | 바이너리 | 30초 |
| [2] | MetalLB + IP Pool | Helm + CRD | 1~2분 |
| [3] | Envoy Gateway + GatewayClass | Helm + CRD | 1~2분 |
| [4] | cert-manager + ClusterIssuer | Helm + CRD | 1~2분 |
| [5] | Headlamp + Admin 계정 | Helm + RBAC | 1~2분 |
| [6] | kube-prometheus-stack | Helm | 2~3분 |
| [7] | Loki + Alloy | Helm | 2~3분 |
| [8] | Sealed Secrets + kubeseal CLI | Helm + 바이너리 | 1분 |
| [9] | Argo CD | Manifest (SSA) | 1~2분 |
| [10] | Image Updater | Manifest | 30초 |

### 완료 시 출력되는 접속 정보

스크립트 마지막에 모든 서비스의 접속 방법이 출력됩니다. 꼭 확인하세요.

## 4단계 — 설치 확인

### 노드 상태 확인

```bash
kubectl get nodes -o wide
# NAME          STATUS   ROLES                  AGE   VERSION
# k3s-master    Ready    control-plane,master   15m   v1.32.x+k3s1
```

### 전체 Pod 상태 확인

```bash
kubectl get pods -A
# 모든 Pod가 Running 또는 Completed 상태여야 함
```

### 네임스페이스별 확인

```bash
kubectl get all -n metallb-system        # MetalLB
kubectl get all -n envoy-gateway-system  # Envoy Gateway
kubectl get all -n cert-manager          # cert-manager
kubectl get all -n headlamp              # Headlamp
kubectl get all -n monitoring            # Prometheus + Grafana
kubectl get all -n logging               # Loki + Alloy
kubectl get all -n argocd                # Argo CD + Image Updater
```

### Envoy Gateway External IP 확인

```bash
kubectl get gateway eg -o jsonpath="{.status.addresses[0].value}"
# MetalLB가 할당한 IP가 출력되어야 함
```

## 설치 실패 시 트러블슈팅

### K3s 완전 재설치

```bash
/usr/local/bin/k3s-uninstall.sh
sudo bash k3s-master.sh
```

### Helm 설치 실패 (timeout)

```bash
helm uninstall <릴리스명> -n <네임스페이스>
# k3s-apps.sh에서 해당 단계만 복사해서 수동 실행
```

### Pod가 Pending 상태

```bash
kubectl describe pod <pod-name> -n <namespace>
# Events 섹션에서 원인 확인
# 흔한 원인: 리소스 부족, 이미지 Pull 실패, PV 없음
```

### K3s 서비스 로그 확인

```bash
journalctl -u k3s -f --no-pager
```
