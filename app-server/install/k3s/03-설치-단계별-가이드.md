# 03. K3s 설치 단계별 가이드

## 전체 설치 흐름

```
[사전 작업]  OCI 콘솔 설정 (OCI만 해당)     → 02-사전준비-가이드.md 참고
[1단계]     k3s-common.sh     → OS 설정 (3~5분)
[2단계]     k3s-master.sh     → K3s Server 설치 (1~2분)
[3단계]     k3s-apps.sh       → 전체 DevOps 스택 설치 (10~15분)
[4단계]     확인 및 접속       → kubectl + 브라우저 접속
```

총 소요 시간: 약 15~25분 (인터넷 속도에 따라 다름, OCI 콘솔 작업 제외)

## [OCI 전용] 스크립트 실행 전 Pre-flight 체크

**OCI 환경이라면 스크립트 실행 전 아래를 반드시 확인하세요.**
일반 서버는 이 섹션을 건너뜁니다.

```bash
# 1. OCI 콘솔에서 확인 (브라우저)
#    - VCN Internet Gateway: Active 상태
#    - 라우트 테이블: 0.0.0.0/0 → Internet Gateway 룰 존재
#    - Security List 인바운드: 22, 6443, 30080, 10250 포트 오픈

# 2. Public IP 확인 (OCI 콘솔 → 인스턴스 → 공용 IP 주소)
#    이 IP를 k3s-common.sh의 PUBLIC_IP에 입력

# 3. SSH 접속 확인
ssh -i ~/.ssh/oci_key opc@<PUBLIC_IP>

# 4. 인터넷 연결 확인 (인스턴스 내부에서)
curl -I https://get.k3s.io
```

## 0단계 — 스크립트 준비

```bash
mkdir -p /opt/k3s-lab
cd /opt/k3s-lab
# 스크립트 파일 복사 후:
chmod +x k3s-*.sh
```

**각 스크립트 최상단 `★ 설정 구간`만 수정합니다 (그 아래는 수정 불필요)**

**k3s-common.sh** — 환경 선택 + IP 설정:
```bash
vi k3s-common.sh
```

| 환경 | 수정할 변수 | 예시 |
|------|-----------|------|
| 일반 서버 | `IS_OCI=false`, `MASTER_IP` | `MASTER_IP="192.168.1.10"` |
| OCI | `IS_OCI=true`, `PUBLIC_IP` | `PUBLIC_IP="150.xxx.xxx.xxx"` |

> `PRIVATE_IP`(OCI)는 자동 감지되므로 수정 불필요합니다.
> `MASTER_IP`(일반)와 `PUBLIC_IP`(OCI)는 다른 개념입니다:
> - `MASTER_IP` → 서버 NIC에 직접 할당된 IP, k3s가 바인딩
> - `PUBLIC_IP` → OCI IGW NAT IP, NIC에 없어 바인딩 불가, `--node-external-ip`로만 사용

**k3s-apps.sh** — 일반 서버라면 MetalLB IP 대역도 수정합니다:
```bash
vi k3s-apps.sh
# ★ 설정 구간 (파일 최상단)
METALLB_IP_START="192.168.1.200"   # ← 서브넷 내 미사용 IP 대역 시작
METALLB_IP_END="192.168.1.220"     # ← 서브넷 내 미사용 IP 대역 끝
```

## 1단계 — OS 기본 설정

마스터 서버에 root로 접속하여 실행합니다.

```bash
sudo bash k3s-common.sh
```

완료 시 `공통 설정 완료!` 메시지와 IP 정보가 출력됩니다.

### k3s-common.sh가 하는 일

| 단계 | 내용 | 일반 서버 | OCI |
|------|------|---------|-----|
| 환경변수 | IP 설정 | `MASTER_IP` 직접 입력 | `PRIVATE_IP` 자동 감지 + `PUBLIC_IP` 검증 |
| [0-1] | root 비밀번호 | `root:1234` 설정 | 동일 |
| [0-2] | 타임존 | Asia/Seoul, NTP | 동일 |
| [0-3] | 기본 패키지 | dnf-utils, iproute-tc, git, curl | + `iptables-services` 추가 |
| [0-4] | /etc/hosts | `MASTER_IP` → 호스트명 | `PRIVATE_IP` → 호스트명 |
| [0-5] | 방화벽 | firewalld 비활성화 | + **iptables REJECT 룰 제거** + ACCEPT 정책 + 영구 저장 |
| [0-6] | Swap 비활성화 | kubelet 요구사항 | 동일 |
| [0-7] | SELinux | Permissive 모드 | 동일 |
| [1] | iptables 모듈 | overlay, br_netfilter + sysctl | 동일 |
| [2] | 한글 | glibc-langpack-ko | 동일 |
| [3] | 환경변수 저장 | `/etc/k3s-env`에 `MASTER_IP` | `/etc/k3s-env`에 `PRIVATE_IP` + `PUBLIC_IP` |

> K8s와 달리 containerd, kubeadm 설치 단계가 없습니다.
> K3s가 containerd를 자동으로 포함합니다.

## 2단계 — K3s Server 설치

```bash
sudo bash k3s-master.sh
```

### k3s-master.sh가 하는 일

| 단계 | 내용 | 일반 서버 | OCI |
|------|------|---------|-----|
| 환경변수 로드 | `/etc/k3s-env` 읽기 | `MASTER_IP` | `PRIVATE_IP`, `PUBLIC_IP` |
| [1] | K3s 설치 옵션 | `--node-ip MASTER_IP`<br>`--tls-san MASTER_IP` | `--node-ip PRIVATE_IP`<br>`--advertise-address PRIVATE_IP`<br>`--node-external-ip PUBLIC_IP`<br>`--tls-san PUBLIC_IP`<br>`--tls-san PRIVATE_IP` |
| [2] | kubectl 설정 | kubeconfig 복사 + KUBECONFIG 환경변수 | 동일 |
| [3] | kubectl 편의 | bash 자동완성, `alias k=kubectl` | 동일 |
| [4] | 확인 | `kubectl get nodes -o wide` | 동일 |

### 완료 확인

```bash
kubectl get nodes -o wide
# NAME          STATUS   ROLES                  AGE   VERSION   INTERNAL-IP    EXTERNAL-IP
# k3s-master    Ready    control-plane,master   1m    v1.32.x   <PRIVATE_IP>   <PUBLIC_IP>
```

> OCI 환경에서 `EXTERNAL-IP` 컬럼에 Public IP가 표시되면 정상입니다.

## 4단계 — 설치 확인

### 노드 상태

```bash
kubectl get nodes -o wide
# STATUS가 Ready, OCI면 EXTERNAL-IP에 Public IP가 보여야 함
```

### 전체 Pod 상태

```bash
kubectl get pods -A
# 모든 Pod가 Running 또는 Completed 상태여야 함
# Pending이 있으면 describe로 원인 확인
kubectl describe pod <pod-name> -n <namespace>
```

### 네임스페이스별 확인

```bash
kubectl get all -n metallb-system        # MetalLB (일반 서버만)
kubectl get all -n envoy-gateway-system  # Envoy Gateway
kubectl get all -n cert-manager          # cert-manager
kubectl get all -n headlamp              # Headlamp
kubectl get all -n monitoring            # Prometheus + Grafana
kubectl get all -n logging               # Loki + Alloy
kubectl get all -n argocd                # Argo CD + Image Updater
```

### Envoy Gateway 확인

```bash
# 일반 서버: LoadBalancer External IP 확인
kubectl get gateway eg -o jsonpath="{.status.addresses[0].value}"

# OCI: NodePort 30080 확인
kubectl get svc -n envoy-gateway-system
# TYPE이 NodePort이고, 30080 포트가 있어야 함
```

## [OCI 전용] 설치 후 추가 작업

### kubeconfig 원격 접속 설정 (로컬 PC에서 kubectl 사용)

서버의 kubeconfig는 기본적으로 Private IP를 서버 주소로 사용합니다.
로컬 PC에서 kubectl로 접속하려면 Public IP로 변경이 필요합니다.

**서버에서 kubeconfig 추출:**
```bash
# 서버에서 실행 — Public IP로 server 주소 변경해서 출력
cat /etc/rancher/k3s/k3s.yaml | \
  sed "s/127.0.0.1/$(cat /etc/k3s-env | grep PUBLIC_IP | cut -d= -f2)/g"
```

**로컬 PC에서 설정:**
```bash
# 위 출력 내용을 ~/.kube/config에 붙여넣거나:
mkdir -p ~/.kube
scp -i ~/.ssh/oci_key opc@<PUBLIC_IP>:/etc/rancher/k3s/k3s.yaml ~/.kube/config
# 그 다음 server 주소를 Public IP로 변경
sed -i "s/127.0.0.1/<PUBLIC_IP>/g" ~/.kube/config

# 접속 확인
kubectl get nodes
```

> OCI Security List에서 6443/tcp가 열려 있어야 원격 접속이 됩니다.

### OCI 인스턴스 재부팅 후 확인사항

OCI 인스턴스 재시작 후 아래를 확인합니다.

```bash
# k3s 서비스 자동 시작 확인
systemctl status k3s

# iptables 룰 유지 확인 (REJECT 룰이 없어야 함)
iptables -L INPUT -n | grep REJECT

# iptables 서비스 활성화 확인 (k3s-common.sh에서 enable됨)
systemctl status iptables
```

> Ephemeral Public IP를 사용하는 경우, 인스턴스 재시작 시 Public IP가 바뀔 수 있습니다.
> 바뀌면 k3s-common.sh를 다시 실행하고, kubeconfig와 hosts 파일을 갱신해야 합니다.
> **Reserved IP를 사용하면 이 문제가 없습니다.**

## 설치 실패 시 트러블슈팅

### K3s 완전 재설치

```bash
/usr/local/bin/k3s-uninstall.sh
sudo bash k3s-master.sh
```

### Helm 설치 실패 (timeout)

```bash
# 실패한 릴리스 삭제 후 수동으로 해당 단계 재실행
helm uninstall <릴리스명> -n <네임스페이스>
# k3s-apps.sh에서 해당 단계만 복사해서 수동 실행
```

### Pod가 Pending 상태

```bash
kubectl describe pod <pod-name> -n <namespace>
# Events 섹션에서 원인 확인
# 흔한 원인: 리소스 부족, 이미지 Pull 실패, PV 없음
```

### K3s 서비스 로그

```bash
journalctl -u k3s -f --no-pager
```

### [OCI] Flannel Pod 간 통신 안 될 때

```bash
# VXLAN 포트(8472/UDP) 확인 — OCI Security List에서도 열려 있어야 함
iptables -L | grep FORWARD
# FORWARD 체인이 ACCEPT여야 함
iptables -P FORWARD ACCEPT
```
