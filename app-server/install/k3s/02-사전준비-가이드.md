# 02. 사전 준비 가이드

## 서버 요구사항

| 항목 | 최소 | 권장 |
|------|------|------|
| OS | Rocky Linux 9 / RHEL 9 / CentOS 9 Stream | Rocky Linux 9 |
| CPU | 2 코어 | 4 코어 이상 |
| RAM | 4 GB | 8 GB 이상 |
| Disk | SSD 40 GB | SSD 100 GB 이상 |
| 네트워크 | 고정 IP | - |

> K3s 자체는 512MB RAM으로도 동작하지만,
> DevOps 스택(Prometheus, Grafana, Loki, ArgoCD 등)을 함께 실행하려면 최소 4GB가 필요합니다.

---

## 네트워크 요구사항

### 필수 포트

| 포트 | 프로토콜 | 용도 |
|------|---------|------|
| 6443 | TCP | K3s API Server |
| 10250 | TCP | kubelet |
| 80 | TCP | HTTP (Envoy Gateway) |
| 443 | TCP | HTTPS (Envoy Gateway) |

> 스크립트에서 firewalld를 비활성화하므로,
> 외부 방화벽(AWS Security Group, 물리 방화벽 등)에서 위 포트를 열어야 합니다.

### MetalLB IP 대역

MetalLB가 서비스에 할당할 External IP 범위를 미리 계획해야 합니다.

| 항목 | 설명 |
|------|------|
| 대역 | 서버와 같은 서브넷 내에서 사용하지 않는 IP 범위 |
| 기본값 | `192.168.1.200 ~ 192.168.1.220` (스크립트 환경변수에서 변경) |
| 주의 | 서버 IP, DHCP 범위와 겹치지 않아야 함 |

## 네트워크 구성

```
클라이언트 PC (브라우저)
    │
    └── 네트워크 (서버와 같은 서브넷 또는 라우팅 가능)
            │
            ├── k3s-master (192.168.1.10)
            │       ├── K3s Server (API + 워크로드)
            │       └── 모든 Pod 실행
            │
            └── MetalLB Pool (192.168.1.200~220)
                    │
                    └── Envoy Gateway External IP
```

## 파일 배치

서버의 원하는 디렉토리에 스크립트 파일을 배치합니다.

```
/opt/k3s-lab/
├── k3s-common.sh
├── k3s-master.sh
├── k3s-apps.sh
└── (문서들)
```

### 파일 전송 방법

**방법 1 — scp로 전송**
```bash
scp k3s-*.sh root@192.168.1.10:/opt/k3s-lab/
```

**방법 2 — Git으로 클론**
```bash
git clone <저장소URL> /opt/k3s-lab
```

**방법 3 — 직접 생성**
```bash
mkdir -p /opt/k3s-lab
vi /opt/k3s-lab/k3s-common.sh
# 스크립트 내용 붙여넣기
```

## 환경 변수 확인 (실행 전 필수!)

각 스크립트 상단에 환경변수가 있습니다. **실행 전에 서버 환경에 맞게 수정하세요.**

### k3s-common.sh

```bash
MASTER_IP="192.168.1.10"        # 마스터 노드 IP
MASTER_HOSTNAME="k3s-master"    # 마스터 호스트명
```

### k3s-master.sh

```bash
MASTER_IP="192.168.1.10"        # 마스터 노드 IP
```

### k3s-apps.sh

```bash
METALLB_IP_START="192.168.1.200"  # MetalLB IP 시작
METALLB_IP_END="192.168.1.220"    # MetalLB IP 끝
```

## hosts 파일 설정 (서비스 접속용)

서비스 설치 후, 접속할 PC의 hosts 파일에 도메인을 추가합니다.
Gateway IP는 설치 완료 후 확인 가능하며, MetalLB 풀의 첫 번째 IP가 보통 할당됩니다.

### Linux/Mac

```bash
sudo bash -c 'cat >> /etc/hosts << EOF
192.168.1.200  grafana.local
192.168.1.200  prometheus.local
192.168.1.200  argocd.local
192.168.1.200  dashboard.local
EOF'
```

### Windows

`C:\Windows\System32\drivers\etc\hosts` 파일을 관리자 권한 메모장으로 열고 추가:

```
192.168.1.200  grafana.local
192.168.1.200  prometheus.local
192.168.1.200  argocd.local
192.168.1.200  dashboard.local
```

## 트러블슈팅

### K3s 설치 실패

```bash
# K3s 서비스 로그 확인
journalctl -u k3s -f

# K3s 완전 삭제 후 재설치
/usr/local/bin/k3s-uninstall.sh
sudo bash k3s-master.sh
```

### 방화벽 관련

```bash
systemctl status firewalld
# 비활성화 안 된 경우
systemctl stop firewalld && systemctl disable firewalld
```

### DNS 확인

```bash
nslookup get.k3s.io
cat /etc/resolv.conf
```

### SELinux 관련

```bash
getenforce    # Permissive 또는 Disabled여야 함
setenforce 0  # 즉시 변경
```
