# 02. 사전 준비 가이드

## 서버 요구사항

| 항목   | 최소                                       | 권장            |
| ---- | ---------------------------------------- | ------------- |
| OS   | Rocky Linux 9 / RHEL 9 / CentOS 9 Stream | Rocky Linux 9 |
| CPU  | 2 코어                                     | 4 코어 이상       |
| RAM  | 4 GB                                     | 8 GB 이상       |
| Disk | SSD 40 GB                                | SSD 100 GB 이상 |
| 네트워크 | 고정 IP (일반 서버) / OCI 인스턴스                 | -             |

> DevOps 스택(Prometheus, Grafana, Loki, ArgoCD 등)을 함께 실행하려면 최소 4GB가 필요합니다.

## ★ 환경 선택 전 확인사항

스크립트를 실행하기 전에 `k3s-common.sh` 상단의 `IS_OCI` 값만 설정하면 됩니다.

```bash
IS_OCI=false   # 일반 서버/VM (온프레미스, AWS, GCP 등) ← 기본값
IS_OCI=true    # OCI (Oracle Cloud Infrastructure)
```

환경에 따라 사전 준비 작업이 다릅니다. 아래에서 해당 항목을 읽으세요.

## [일반 서버] 사전 준비

### 수정할 환경변수

`k3s-common.sh`와 `k3s-apps.sh`에서 아래 두 곳만 수정합니다.

**k3s-common.sh**
```bash
MASTER_IP="192.168.1.10"     # ← 마스터 노드 실제 IP로 변경
MASTER_HOSTNAME="k3s-master" # ← 원하는 호스트명
```

**k3s-apps.sh**
```bash
METALLB_IP_START="192.168.1.200"  # ← 서브넷 내 미사용 IP 범위 시작
METALLB_IP_END="192.168.1.220"    # ← 서브넷 내 미사용 IP 범위 끝
```

> MetalLB IP 대역은 서버 IP, DHCP 범위와 겹치지 않아야 합니다.

### 네트워크 요구사항 (일반 서버)

외부 방화벽/라우터에서 아래 포트가 열려 있어야 합니다.

| 포트 | 프로토콜 | 용도 |
|------|---------|------|
| 6443 | TCP | K3s API Server |
| 80 | TCP | HTTP (Envoy Gateway) |
| 443 | TCP | HTTPS |
| 8472 | UDP | Flannel VXLAN (멀티 노드 시) |
| 10250 | TCP | kubelet |

## [OCI] 사전 준비 — 스크립트 실행 전 반드시 완료

OCI 환경은 클라우드 콘솔 작업이 추가로 필요합니다.
**아래 순서대로 진행하지 않으면 k3s 설치 후에도 외부 접속이 되지 않습니다.**

---

### OCI 네트워크 구조 이해

OCI 인스턴스는 NIC에 Private IP만 할당되고, Public IP는 OCI Internet Gateway에서 NAT로 매핑됩니다.

```
외부 클라이언트
      ↓
  Public IP (OCI Internet Gateway — NAT)
      ↓ 1:1 NAT
  Private IP (실제 NIC: eth0 또는 enp0s3)
      ↓
   k3s 노드 (Private IP로만 바인딩)
```

이 구조 때문에 k3s에 Public IP를 직접 쓰면 바인딩 에러가 발생합니다.
스크립트는 이 구조를 자동으로 처리합니다.

---

### STEP 1 — OCI 인스턴스 생성 체크리스트

인스턴스 생성 시 아래 항목을 확인합니다.

| 항목 | 권장값 | 이유 |
|------|--------|------|
| OS 이미지 | Rocky Linux 9 (Marketplace) | 이 스크립트의 대상 OS |
| Shape | VM.Standard.A1.Flex (ARM, Free Tier) | 무료, 최대 4 OCPU / 24GB RAM |
| OCPU | 2 이상 | DevOps 스택 실행 최소 |
| Memory | 8 GB 이상 | Prometheus, Loki 포함 시 |
| Boot Volume | 50 GB 이상 | 컨테이너 이미지 저장 공간 |
| Public IP 할당 | **Ephemeral 또는 Reserved** | 반드시 Public IP 있어야 함 |
| VCN | 기존 VCN 또는 신규 생성 | Internet Gateway가 연결된 VCN |
| Subnet | Public Subnet | Public IP 라우팅 필요 |
| SSH 키 | 본인 공개키 등록 | 인스턴스 접속용 |

> **Reserved IP 권장**: Ephemeral IP는 인스턴스 재시작 시 바뀔 수 있습니다.
> 재시작해도 IP가 바뀌지 않으려면 Reserved Public IP를 할당하세요.

---

### STEP 2 — VCN Internet Gateway 확인

인스턴스가 인터넷에 연결되려면 VCN에 Internet Gateway가 붙어 있어야 합니다.

**OCI 콘솔 → 네트워킹 → Virtual Cloud Networks → VCN 선택**

1. **인터넷 게이트웨이** 탭 클릭
2. Internet Gateway가 **Active** 상태인지 확인
3. 없으면 **인터넷 게이트웨이 생성** 클릭

**라우트 테이블 확인**

1. VCN → **라우트 테이블** → 기본 라우트 테이블 클릭
2. 아래 룰이 있어야 함:

| 대상 유형 | 대상 | 설명 |
|---------|------|------|
| Internet Gateway | `0.0.0.0/0` | 모든 외부 트래픽 → IGW |

없으면 **라우트 규칙 추가**로 위 내용 추가합니다.

---

### STEP 3 — Security List 인바운드 포트 오픈 (필수!)

**OCI 콘솔 → 네트워킹 → VCN → 보안 → Security List → 보안 규칙 → 수신규칙**

아래 포트를 **모두** 추가합니다.

| 소스 CIDR     | 프로토콜 | 포트    | 용도                              | 필수 여부   |
| ----------- | ---- | ----- | ------------------------------- | ------- |
| `0.0.0.0/0` | TCP  | 22    | SSH 접속                          | 필수      |
| `0.0.0.0/0` | TCP  | 6443  | k3s API Server (원격 kubectl)     | 필수      |
| `0.0.0.0/0` | TCP  | 30080 | Envoy Gateway NodePort (HTTP)   | 필수      |
| `0.0.0.0/0` | TCP  | 443   | HTTPS (TLS 설정 시)                | 권장      |
| `0.0.0.0/0` | TCP  | 10250 | kubelet                         | 워커 추가 시 |
| `0.0.0.0/0` | UDP  | 8472  | Flannel VXLAN (멀티 노드 시)         | 워커 추가 시 |
| `0.0.0.0/0` | UDP  | 51820 | WireGuard/Flannel 암호화 (멀티 노드 시) | 워커 추가 시 |

**추가 방법:**
1. 수신 규칙 → **수신 규칙 추가** 클릭
2. 소스 유형: CIDR, 소스 CIDR: `0.0.0.0/0`
3. IP 프로토콜: TCP (또는 UDP)
4. 대상 포트 범위: 포트 번호 입력
5. **수신 규칙 추가** 클릭

> NSG(Network Security Group)를 사용하는 경우에도 동일한 포트를 인바운드 허용합니다.

---

### STEP 4 — OCI 인스턴스 내부 iptables 확인

OCI 기본 이미지에는 iptables에 REJECT 룰이 기본으로 설정되어 있습니다.
**k3s-common.sh가 이를 자동으로 처리하지만**, 스크립트 실행 전 현재 상태를 확인해두면 좋습니다.

```bash
# SSH로 인스턴스 접속 후 실행
sudo iptables -L INPUT -n --line-numbers | grep REJECT
# REJECT 룰이 보이면 스크립트가 제거함 (정상)
```

---

### STEP 5 — 수정할 환경변수 (OCI)

**k3s-common.sh에서 아래 두 곳만 수정합니다.**

```bash
IS_OCI=true                   # OCI 환경 활성화

PUBLIC_IP="150.xxx.xxx.xxx"   # ← OCI 콘솔에서 확인한 Public IP 입력
MASTER_HOSTNAME="k3s-master"  # ← 호스트명 (선택, 변경 가능)
```

> Private IP는 자동 감지되므로 입력하지 않아도 됩니다.
> Public IP: OCI 콘솔 → 인스턴스 → 인스턴스 정보 → **공용 IP 주소**

**다른 스크립트(k3s-master.sh, k3s-apps.sh)는 수정하지 않아도 됩니다.**
`/etc/k3s-env`를 통해 IP 값을 자동으로 공유합니다.

## 파일 배치

서버의 원하는 디렉토리에 스크립트 파일을 배치합니다.

```
/opt/k3s-lab/
├── k3s-common.sh
├── k3s-master.sh
├── k3s-apps.sh
└── (문서들)
```

### 파일 전송 방법

**방법 1 — scp로 전송**
```bash
# 일반 서버
scp k3s-*.sh root@192.168.1.10:/opt/k3s-lab/

# OCI (SSH 키 인증)
scp -i ~/.ssh/oci_key k3s-*.sh opc@<PUBLIC_IP>:/opt/k3s-lab/
```

**방법 2 — Git으로 클론**
```bash
git clone <저장소URL> /opt/k3s-lab
```

**방법 3 — 직접 생성**
```bash
mkdir -p /opt/k3s-lab
vi /opt/k3s-lab/k3s-common.sh
# 스크립트 내용 붙여넣기
```

## hosts 파일 설정 (서비스 접속용)

설치 완료 후, **접속할 PC**의 hosts 파일에 도메인을 추가합니다.

### 일반 서버

```bash
# <GATEWAY_IP>를 MetalLB가 할당한 IP로 변경 (설치 후 스크립트 출력에서 확인)
sudo bash -c 'cat >> /etc/hosts << EOF
<GATEWAY_IP>  grafana.local
<GATEWAY_IP>  prometheus.local
<GATEWAY_IP>  argocd.local
<GATEWAY_IP>  dashboard.local
EOF'
```

### OCI

```bash
# <OCI_PUBLIC_IP>를 실제 Public IP로 변경
sudo bash -c 'cat >> /etc/hosts << EOF
<OCI_PUBLIC_IP>  grafana.local
<OCI_PUBLIC_IP>  prometheus.local
<OCI_PUBLIC_IP>  argocd.local
<OCI_PUBLIC_IP>  dashboard.local
EOF'
```

**Windows** — `C:\Windows\System32\drivers\etc\hosts` (관리자 권한 메모장)
```
<IP>  grafana.local
<IP>  prometheus.local
<IP>  argocd.local
<IP>  dashboard.local
```

## 트러블슈팅

### K3s 설치 실패

```bash
journalctl -u k3s -f              # 실시간 로그
/usr/local/bin/k3s-uninstall.sh   # 완전 삭제 후 재설치
sudo bash k3s-master.sh
```

### [OCI] 외부 접속 불가

```bash
# 1. OCI Security List 포트 확인 (OCI 콘솔에서 확인)

# 2. iptables REJECT 룰 확인
sudo iptables -L INPUT -n | grep REJECT
# 남아있으면:
sudo iptables -P INPUT ACCEPT && sudo iptables -F INPUT
sudo iptables-save > /etc/sysconfig/iptables

# 3. Envoy Gateway NodePort 확인
kubectl get svc -n envoy-gateway-system
# TYPE이 NodePort, 포트 30080이어야 함

# 4. 포트 리스닝 확인
ss -tlnp | grep 30080
```

### 방화벽 관련

```bash
systemctl status firewalld
systemctl stop firewalld && systemctl disable firewalld
```

### DNS 확인

```bash
nslookup get.k3s.io
cat /etc/resolv.conf
```

### SELinux 관련

```bash
getenforce    # Permissive 또는 Disabled여야 함
setenforce 0
```
