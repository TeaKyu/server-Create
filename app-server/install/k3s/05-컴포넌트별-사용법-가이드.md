## 1. K3s 서버 재반영

### NAT 미사용

```bash
source /etc/k3s-env
curl -sfL https://get.k3s.io | sh -s - server \
  --node-ip "$DIRECT_BIND_IP" \
  --tls-san "$DIRECT_BIND_IP" \
  --disable traefik \
  --disable servicelb \
  --write-kubeconfig-mode 644
```

### NAT 사용, 외부 IP 미확인

```bash
source /etc/k3s-env
curl -sfL https://get.k3s.io | sh -s - server \
  --node-ip "$PRIVATE_IP" \
  --advertise-address "$PRIVATE_IP" \
  --tls-san "$PRIVATE_IP" \
  --disable traefik \
  --disable servicelb \
  --write-kubeconfig-mode 644
```

### NAT 사용, 외부 IP 확인

```bash
source /etc/k3s-env
curl -sfL https://get.k3s.io | sh -s - server \
  --node-ip "$PRIVATE_IP" \
  --advertise-address "$PRIVATE_IP" \
  --node-external-ip "$EXTERNAL_IP" \
  --tls-san "$PRIVATE_IP" \
  --tls-san "$EXTERNAL_IP" \
  --disable traefik \
  --disable servicelb \
  --write-kubeconfig-mode 644
```

## 3. Envoy Gateway

### 동작 방식

| 환경 | 서비스 타입 | 접속 방식 |
|------|-------------|----------|
| NAT 미사용 | LoadBalancer | `http://도메인` |
| NAT 사용 | NodePort | `http://도메인:30080` |

확인:

```bash
kubectl get svc -n envoy-gateway-system
kubectl get httproute
```

NAT 사용 환경에서 자동 패치가 실패했다면:

```bash
kubectl patch svc <envoy-서비스명> -n envoy-gateway-system --type merge \
  -p '{"spec":{"type":"NodePort","ports":[{"name":"http","port":80,"targetPort":80,"nodePort":30080,"protocol":"TCP"}]}}'
```

## 9. 외부 IP를 나중에 반영하는 절차

NAT 사용 환경에서 처음엔 external IP 없이 설치했다가 나중에 알게 된 경우:

1. `/etc/k3s-env`의 `EXTERNAL_IP` 값을 업데이트
2. K3s server를 external IP 포함 옵션으로 재반영
3. 워커가 있다면 worker도 external IP 포함 옵션으로 재반영
4. kubeconfig `server:` 주소를 `https://EXTERNAL_IP:6443`로 갱신
5. hosts 파일과 접속 URL 안내를 external IP 기준으로 갱신

이 과정은 사용자 서비스 구조를 바꾸는 작업이 아니라 외부 진입 주소를 완성하는 작업이야.
상세 절차는 [07-외부-IP-나중-반영-가이드.md](/Users/taekyu/Documents/git/server-Create/app-server/install/k3s/07-외부-IP-나중-반영-가이드.md) 참고.

## 11. 전체 삭제

현재 노드의 K3s와 로컬 설정을 정리하려면:

```bash
sudo bash k3s-cleanup.sh
```

확인 프롬프트 없이 실행:

```bash
sudo bash k3s-cleanup.sh --yes
```

`helm`, `kubeseal`까지 같이 삭제:

```bash
sudo bash k3s-cleanup.sh --yes --remove-tools
```

옵션 차이:

- 기본 실행
  현재 노드의 K3s와 로컬 설정을 지우고, 삭제 전에 확인 질문을 한 번 묻는다.
- `--yes`
  확인 질문 없이 바로 진행한다.
- `--remove-tools`
  `helm`, `kubeseal`도 같이 삭제한다.

주의:

- 이 스크립트는 현재 노드만 정리한다
- 워커가 있는 클러스터면 워커를 먼저 drain / delete 한 뒤 각 워커에서 실행해야 한다
- SELinux, firewalld, root 비밀번호 같은 OS 설정은 자동 복구하지 않는다
