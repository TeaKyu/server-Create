# 01. K3s 프로젝트 개요 및 아키텍처

## 프로젝트 목적

실제 서버(베어메탈/VM) 환경에서 Rocky Linux 9 기반 K3s 경량 Kubernetes 클러스터를 구축하고,
운영에 필요한 전체 DevOps 스택을 설치하여 실제 운영 가능한 인프라를 만드는 것이 목표입니다.

마스터 1대로 모든 것을 실행하는 구성이며, 워커 노드 확장이 필요하면 `06-워커-노드-확장-가이드.md`를 참고하세요.

---

## K8s(kubeadm) vs K3s 비교

| 항목 | K8s (kubeadm) | K3s (현재) |
|------|--------------|------------|
| 설치 방식 | containerd → kubeadm init | `curl` 한 줄로 설치 |
| 바이너리 크기 | 약 300MB+ (여러 컴포넌트) | 약 60MB (단일 바이너리) |
| CNI (Pod 네트워크) | Calico 별도 설치 | Flannel 내장 |
| 컨테이너 런타임 | containerd 별도 설치 | containerd 내장 |
| etcd | etcd 클러스터 | SQLite (단일 서버) |
| 기본 Ingress | 없음 | Traefik 내장 (비활성화 가능) |
| 기본 LB | 없음 | ServiceLB 내장 (비활성화 가능) |
| kubeconfig | `$HOME/.kube/config` | `/etc/rancher/k3s/k3s.yaml` |
| 리소스 요구 | Master 최소 2GB RAM | Server 최소 512MB RAM |

---

## 클러스터 구성

| 노드 | 호스트명 | IP | 최소 사양 | 역할 |
|------|---------|-----|----------|------|
| Master | k3s-master | 192.168.1.10 | 2 core / 4 GB | Server + 워크로드 실행 |

> K3s는 마스터에 기본적으로 워크로드 스케줄링을 허용합니다.
> kubeadm과 달리 taint가 없으므로 마스터 1대로 모든 Pod를 실행할 수 있습니다.

## 전체 스택 구성

```
┌─────────────────────────────────────────────────────────────┐
│                    외부 트래픽 (브라우저)                       │
└──────────────────────────┬──────────────────────────────────┘
                           │
                    ┌──────▼──────┐
                    │   MetalLB   │  External IP 할당
                    └──────┬──────┘
                           │
                 ┌─────────▼─────────┐
                 │  Envoy Gateway    │  Gateway API (HTTPRoute) 라우팅
                 └─────────┬─────────┘
                           │
          ┌────────────────┼────────────────┐
          │                │                │
    ┌─────▼─────┐   ┌─────▼─────┐   ┌─────▼─────┐
    │  Service A │   │  Service B │   │  Service C │
    └───────────┘   └───────────┘   └───────────┘

┌─────────────────────────────────────────────────────────────┐
│  인프라 도구                                                   │
│                                                              │
│  [네트워크]    Flannel (K3s 내장 CNI)                          │
│  [인증서]     cert-manager (TLS 자동 발급)                     │
│  [패키지]     Helm 4.x                                        │
│  [모니터링]   Prometheus + Grafana (kube-prometheus-stack)     │
│  [로깅]      Loki + Alloy                                  │
│  [배포]      Argo CD + Image Updater                          │
│  [시크릿]    Sealed Secrets (kubeseal)                         │
│  [UI]        Headlamp (클러스터 웹 UI)                          │
└─────────────────────────────────────────────────────────────┘
```

## 설치 순서 및 의존성

스크립트는 **3개 파일**로 구성됩니다.

```
[k3s-common.sh]   → OS 설정 (방화벽, Swap, SELinux, iptables)
[k3s-master.sh]   → K3s Server 설치 (Flannel 자동 포함)
[k3s-apps.sh]     → 전체 DevOps 스택 설치
```

### 스택 설치 순서 (k3s-apps.sh 내부)

```
[1] Helm            → 이후 모든 Helm 설치의 전제조건
[2] MetalLB         → Envoy Gateway가 External IP를 받으려면 필요
[3] Envoy Gateway   → MetalLB에서 IP를 받아 Gateway API로 트래픽 수신
[4] cert-manager    → Gateway에 TLS를 걸려면 필요
[5] Headlamp        → Helm 필요
[6] Prometheus      → Helm 필요
[7] Loki            → Helm 필요, Grafana에서 데이터소스 연동
[8] Sealed Secrets  → Helm 필요
[9] Argo CD         → Manifest 설치 (Helm 불필요)
[10] Image Updater  → Argo CD가 먼저 설치되어야 함
```

## 버전 호환성 매트릭스

## 파일 목록

| 파일 | 용도 |
|-----|------|
| `k3s-common.sh` | OS 설정 (방화벽, Swap, SELinux, iptables) |

## K8s 원본 대비 변경 사항

| 항목 | K8s (원본) | K3s (현재) |
|------|-----------|------------|
| 클러스터 도구 | kubeadm + kubelet + kubectl | K3s (단일 바이너리) |
| 컨테이너 런타임 | containerd 별도 설치 | K3s 내장 |
| CNI | Calico (별도 Manifest) | Flannel (K3s 내장) |
| 기본 Ingress | 없음 | Traefik (비활성화 → Envoy Gateway) |
| 기본 LB | 없음 | ServiceLB (비활성화 → MetalLB) |
| VM 관리 | Vagrant + VirtualBox | 실제 서버 (베어메탈/VM) |
| 노드 구성 | Master 1 + Worker 2 | Master 1대 단독 |
| kubeconfig | `$HOME/.kube/config` | `/etc/rancher/k3s/k3s.yaml` |
| 스크립트 수 | 5개 (Vagrantfile 포함) | 3개 |

## 운영 환경 전환 시 변경 포인트

**Helm 차트가 동일하므로 values.yaml만 교체하면 됩니다.**

| 컴포넌트 | 현재 (단일 서버) | 운영 변경 |
|---------|---------------|----------|
| **K3s** | Server 1대 (SQLite) | Server 3대 + embedded etcd (HA) |
| **Loki** | SingleBinary + emptyDir | SingleBinary 2개 + S3/GCS |
| **Prometheus** | retention 7일 | retention + 원격 쓰기 또는 PV 확보 |
| **Grafana** | admin1234 | OIDC/SSO 연동, 비밀번호 변경 |
| **ArgoCD** | insecure 모드 (HTTP) | TLS 활성화 (cert-manager 연동) |
| **Sealed Secrets** | 기본 | 공개키 백업 필수, 키 로테이션 정책 |
| **MetalLB** | 로컬 IP 대역 | 실제 네트워크 IP 대역 |
| **cert-manager** | self-signed | Let's Encrypt 또는 내부 CA |
