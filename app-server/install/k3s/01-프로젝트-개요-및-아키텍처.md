# 01. K3s 프로젝트 개요 및 아키텍처

## 프로젝트 목적

Rocky Linux 9 기반 K3s 경량 Kubernetes 클러스터를 구축하고,
운영에 필요한 전체 DevOps 스택을 설치하여 실제 운영 가능한 인프라를 만드는 것이 목표입니다.

마스터 1대로 모든 것을 실행하는 구성이며, 워커 노드 확장이 필요하면 `06-워커-노드-확장-가이드.md`를 참고하세요.

> **이 스크립트는 두 가지 환경을 지원합니다.**
> `IS_OCI` 플래그 하나로 전환합니다. (기본값: 일반 서버)

---

## 환경별 차이 한눈에 보기

| 항목           | 일반 서버 (기본값)                         | OCI (Oracle Cloud)                                         |
| ------------ | ----------------------------------- | ---------------------------------------------------------- |
| IP 구성        | 고정 IP 1개 직접 입력                      | Private IP(NIC) + Public IP(NAT) 분리                        |
| k3s IP 옵션    | `--node-ip` 1개                      | `--node-ip` + `--advertise-address` + `--node-external-ip` |
| LoadBalancer | MetalLB (L2)                        | 사용 안 함                                                     |
| 외부 접속 방식     | MetalLB External IP → Envoy Gateway | OCI Public IP:30080 → NodePort → Envoy                     |
| 방화벽          | firewalld 비활성화만                     | + iptables REJECT 룰 제거 필요                                  |
| 사전 작업        | 없음                                  | OCI 콘솔 Security List 포트 오픈 필요                              |

## K8s(kubeadm) vs K3s 비교

| 항목 | K8s (kubeadm) | K3s (현재) |
|------|--------------|------------|
| 설치 방식 | containerd → kubeadm init | `curl` 한 줄로 설치 |
| 바이너리 크기 | 약 300MB+ (여러 컴포넌트) | 약 60MB (단일 바이너리) |
| CNI (Pod 네트워크) | Calico 별도 설치 | Flannel 내장 |
| 컨테이너 런타임 | containerd 별도 설치 | containerd 내장 |
| etcd | etcd 클러스터 | SQLite (단일 서버) |
| 기본 Ingress | 없음 | Traefik 내장 (비활성화 → Envoy Gateway) |
| 기본 LB | 없음 | ServiceLB 내장 (비활성화) |
| kubeconfig | `$HOME/.kube/config` | `/etc/rancher/k3s/k3s.yaml` |
| 리소스 요구 | Master 최소 2GB RAM | Server 최소 512MB RAM |

## 클러스터 구성

### 일반 서버

| 노드 | 호스트명 | IP | 최소 사양 | 역할 |
|------|---------|-----|----------|------|
| Master | k3s-master | 192.168.1.10 (직접 입력) | 2 core / 4 GB | Server + 워크로드 실행 |

### OCI

| 노드 | 호스트명 | Private IP | Public IP | 최소 사양 | 역할 |
|------|---------|-----------|-----------|----------|------|
| Master | k3s-master | 자동 감지 (NIC 기준) | OCI 콘솔에서 확인 | 2 core / 4 GB | Server + 워크로드 실행 |

> OCI 인스턴스는 NIC에 Private IP만 할당되고, Public IP는 OCI Internet Gateway에서 NAT로 매핑됩니다.
> k3s는 Private IP 기준으로 바인딩하고, Public IP는 `--node-external-ip`로 별도 지정합니다.

## 설치 순서 및 의존성

스크립트는 **3개 파일**로 구성됩니다.

```
[k3s-common.sh]   → OS 설정 + IS_OCI 플래그에 따른 분기
[k3s-master.sh]   → K3s Server 설치 (Flannel 자동 포함)
[k3s-apps.sh]     → 전체 DevOps 스택 설치
```

### 스택 설치 순서 (k3s-apps.sh 내부)

| 순서 | 컴포넌트 | 설치 방식 | 일반 서버 | OCI |
|------|---------|---------|---------|-----|
| [1] | Helm | curl 바이너리 | 설치 | 설치 |
| [2] | MetalLB + IP Pool | Helm | **설치** | **건너뜀** |
| [3] | Envoy Gateway | Helm | LoadBalancer 타입 | **NodePort 30080 패치** |
| [4] | cert-manager | Helm | 설치 | 설치 |
| [5] | Headlamp | Helm | 설치 | 설치 |
| [6] | Prometheus + Grafana | Helm | 설치 | 설치 |
| [7] | Loki + Alloy | Helm × 2 | 설치 | 설치 |
| [7-2] | Loki 데이터소스 자동 프로비저닝 | kubectl apply | Grafana에 자동 등록 | Grafana에 자동 등록 |
| [8] | Sealed Secrets 컨트롤러 | Helm | 설치 | 설치 |
| [8-1] | kubeseal CLI | Binary (curl) | 설치 | 설치 |
| [9] | Argo CD | Helm | 설치 | 설치 |
| [10] | Image Updater | Helm | 설치 | 설치 |
| [11] | HTTPRoute / ReferenceGrant | kubectl apply | 설정 | 설정 |

> `kubeseal`은 쿠버네티스 리소스가 아닌 CLI 바이너리이므로 Helm 설치 대상이 아닙니다.
> Sealed Secrets **컨트롤러**(클러스터 내 서버 파드)는 Helm으로, `kubeseal` CLI는 binary로 설치합니다.

## 파일 목록

| 파일 | 용도 | 수정 위치 | 수정 내용 |
|-----|------|---------|---------|
| `k3s-common.sh` | OS 설정, 환경 분기 | **파일 최상단 ★ 설정 구간** | `IS_OCI`, `MASTER_HOSTNAME`, `MASTER_IP`(일반), `PUBLIC_IP`(OCI) |
| `k3s-master.sh` | K3s Server 설치 | 수정 불필요 | `/etc/k3s-env` 자동 로드 |
| `k3s-apps.sh` | 전체 DevOps 스택 설치 | **파일 최상단 ★ 설정 구간** | `METALLB_IP_START`, `METALLB_IP_END` (일반 서버만) |
| `k3s-worker.sh` | 워커 노드 조인 | **파일 최상단 ★ 설정 구간** | `K3S_TOKEN`, `MASTER_IP`(일반) 또는 `MASTER_PRIVATE_IP`(OCI) |

> 모든 스크립트의 사용자 입력값은 **파일 최상단의 `★ 설정 구간`에 집중**되어 있습니다.
> 이 구간 아래는 수정하지 않아도 됩니다.

> 워커 노드를 추가하려면 `06-워커-노드-확장-가이드.md`와 `k3s-worker.sh`를 참고하세요.

## 운영 환경 전환 시 변경 포인트

**Helm 차트가 동일하므로 values.yaml만 교체하면 됩니다.**

| 컴포넌트 | 현재 (단일 서버) | 운영 변경 |
|---------|---------------|----------|
| **K3s** | Server 1대 (SQLite) | Server 3대 + embedded etcd (HA) |
| **Loki** | SingleBinary + emptyDir | SingleBinary 2개 + S3/GCS |
| **Prometheus** | retention 7일 | retention + 원격 쓰기 또는 PV 확보 |
| **Grafana** | admin1234 | OIDC/SSO 연동, 비밀번호 변경 |
| **ArgoCD** | insecure 모드 (HTTP) | TLS 활성화 (cert-manager 연동) |
| **Sealed Secrets** | 기본 | 공개키 백업 필수, 키 로테이션 정책 |
| **Envoy (OCI NodePort)** | NodePort 30080 | OCI LB 서비스 또는 도메인 + Let's Encrypt |
| **Envoy (일반 서버)** | MetalLB L2 | BGP 모드 또는 실제 LB 연동 |
| **cert-manager** | self-signed | Let's Encrypt 또는 내부 CA |
