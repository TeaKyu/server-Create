# 04. 서비스 접속 가이드

## 접속 방식: HTTPRoute 도메인 라우팅

모든 서비스는 Envoy Gateway의 HTTPRoute를 통해 도메인으로 접속합니다.
MetalLB가 Gateway에 External IP를 할당하고, Windows hosts 파일에 도메인을 등록하면
브라우저에서 바로 접속할 수 있습니다.

```
브라우저 → hosts 파일 (도메인 → IP) → MetalLB IP → Envoy Gateway → HTTPRoute → Service → Pod
```

port-forward는 필요 없습니다.

---

## 사전 설정: Windows hosts 파일

### Gateway IP 확인 (Master에서)
```bash
kubectl get gateway eg -o jsonpath='{.status.addresses[0].value}'
# 예: 192.168.56.200
```

### hosts 파일 수정

1. 메모장을 **관리자 권한**으로 실행
2. 파일 열기: `C:\Windows\System32\drivers\etc\hosts`
3. 아래 내용 추가 (IP는 위에서 확인한 값):

```
192.168.56.200  grafana.local
192.168.56.200  prometheus.local
192.168.56.200  argocd.local
192.168.56.200  dashboard.local
```

4. 저장 후 브라우저에서 바로 접속 가능

---

## 서비스별 접속 정보

| 서비스 | URL | 인증 |
|--------|-----|------|
| Grafana | http://grafana.local | admin / admin1234 |
| Prometheus | http://prometheus.local | 없음 |
| Argo CD | http://argocd.local | admin / (secret 확인) |
| Headlamp | http://dashboard.local | Bearer Token |

> 모든 서비스가 HTTP(80)로 접속됩니다.
> ArgoCD는 insecure 모드로 설정되어 TLS가 비활성화되어 있습니다.

## Grafana (Prometheus 메트릭 + Loki 로그)

### 접속
브라우저: `http://grafana.local`
ID: `admin` / PW: `admin1234`

### 기본 대시보드 확인
좌측 메뉴 > Dashboards 클릭
이미 다수의 Kubernetes 대시보드가 포함되어 있음:
- Kubernetes / Compute Resources / Cluster — 클러스터 전체 리소스
- Kubernetes / Compute Resources / Namespace (Pods) — 네임스페이스별 Pod 리소스
- Node Exporter / Nodes — 노드별 CPU/메모리/디스크/네트워크

### Loki 데이터소스 추가 (로그 조회)
1. 좌측 메뉴 > Connections > Data Sources
2. Add data source > Loki 선택
3. URL: `http://loki.logging.svc.cluster.local:3100`
4. Save & Test → "Data source successfully connected" 확인

### 로그 조회
좌측 메뉴 > Explore > 상단 데이터소스를 "Loki"로 변경
```
{namespace="default"}                     # 전체 로그
{namespace="argocd", app="argocd-server"} # ArgoCD 로그
{pod="my-app-xxxxx"} |= "error"           # 특정 Pod에서 error 검색
```

## Argo CD

### 접속
브라우저: `http://argocd.local`
(insecure 모드이므로 HTTPS 경고 없음)

### 초기 비밀번호 확인
```bash
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
echo
```
ID: `admin` / PW: 위 명령어 출력값

### 비밀번호 변경 (권장)
```bash
# CLI 설치 후
argocd login argocd.local --insecure --grpc-web
argocd account update-password
```

## Headlamp (클러스터 웹 UI)

### 접속
브라우저: `http://dashboard.local`
(Kubernetes Dashboard의 공식 후속 도구)

### 로그인 토큰 생성
```bash
kubectl -n headlamp create token headlamp-admin
```
출력된 긴 문자열을 복사 → 브라우저 로그인 화면에서 붙여넣기

## Prometheus (직접 쿼리)

### 접속
브라우저: `http://prometheus.local`

### 유용한 PromQL 쿼리
```promql
# 노드별 CPU 사용률
100 - (avg by(instance)(irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# 노드별 메모리 사용률
(1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100

# Pod별 메모리 사용량 (MB)
sum by(pod)(container_memory_working_set_bytes) / 1024 / 1024

# 네임스페이스별 CPU 요청량
sum by(namespace)(kube_pod_container_resource_requests{resource="cpu"})
```

## 트래픽 흐름 이해

```
Windows 브라우저 → http://grafana.local
     │
     │  hosts 파일에서 grafana.local → 192.168.56.200 으로 해석
     │
     ▼
MetalLB (192.168.56.200:80)
     │
     ▼
Envoy Gateway (HTTPRoute 매칭)
     │  Host: grafana.local → backendRef: kube-prometheus-stack-grafana:80
     │
     ▼
Grafana Pod (monitoring 네임스페이스)
```

## HTTPRoute 관리

### 상태 확인
```bash
# 전체 HTTPRoute 목록
kubectl get httproute

# 상세 상태 (Accepted/ResolvedRefs 확인)
kubectl describe httproute grafana
kubectl describe httproute argocd
kubectl describe httproute headlamp
kubectl describe httproute prometheus

# Gateway 상태
kubectl get gateway eg
```

### 내 앱에 HTTPRoute 추가하기
```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: my-app
  namespace: default
spec:
  parentRefs:
  - name: eg
  hostnames:
  - "my-app.local"
  rules:
  - backendRefs:
    - name: my-app-svc
      port: 80
```
적용 후 Windows hosts 파일에 `192.168.56.200  my-app.local` 추가.

## VM 재시작 후

port-forward와 달리 HTTPRoute는 Kubernetes 리소스이므로 **VM 재시작 후에도 유지됩니다.**
별도 재설정이 필요 없습니다.

hosts 파일도 Windows에 저장되어 있으므로 재설정 불필요.
